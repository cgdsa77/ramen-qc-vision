# 抻面标注安全规范与新增视频流程

## 一、当前已标注的 cm 视频数量

**当前已标注**：`cm1`～`cm7`。**本次准备标注**：`cm10`、`cm11`、`cm12`（三段视频，标签与主 classes 一致：hand、noodle_rope、noodle_bundle）。

- 标注根目录：`data/labels/抻面/`
- 主类别定义（**唯一权威**）：`data/labels/抻面/classes.txt`
- 每个视频单独子目录：`data/labels/抻面/cm1/` … `data/labels/抻面/cm12/`，每个目录内有各自的 `classes.txt` 和若干 `*.txt` 标注文件。

**固定类别顺序（不可更改）**：

| 类别 ID | 类别名        | 说明     |
|--------|---------------|----------|
| 0      | hand          | 手部     |
| 1      | noodle_rope   | 面条     |
| 2      | noodle_bundle | 面条束   |

即 `classes.txt` 内容必须**严格按行**为：

```
hand
noodle_rope
noodle_bundle
```

训练与推理都按「0=hand, 1=noodle_rope, 2=noodle_bundle」解析，**顺序一旦不一致，所有已有标注的语义都会错位**。

## 二、为何新标注会破坏之前数据（你遇到过的情况）

常见情况包括：

1. **labelImg 生成了新的 classes.txt**  
   在新视频目录下首次保存时，labelImg 可能按「你第一次使用的类别顺序」生成或覆盖 `classes.txt`。若顺序与主文件不一致（例如先标了 noodle_rope 再标 hand），则新视频里 0=noodle_rope、1=hand，和主文件 0=hand、1=noodle_rope 相反，训练/展示都会错。

2. **改动了主 classes.txt 或其它视频的 classes.txt**  
   若在 labelImg 里「另存为」或编辑了类别列表，并保存到 `data/labels/抻面/classes.txt` 或某个 `cm*/classes.txt`，会把原来的顺序或类别数量改掉，导致所有依赖该文件的标注和训练出错。

3. **新增/删除了类别**  
   若在 labelImg 中增加或删除了类别（例如加了 noodle_rope2、删了 noodle_bundle），会使类别 ID 与主文件不一致，旧标注的 ID 会对应到错误类别。

因此：**新标注操作必须在不改主 classes、不改任何已有 cm 目录内容的前提下进行，且新视频的类别顺序与主文件完全一致。**

## 三、安全新增 2～3 个抻面视频的流程（不影响已有数据）

### 步骤 1：备份（强烈建议）

在新增或标注任何新视频前，先备份标注目录，例如：

```bash
# 在项目根目录执行，生成带日期的备份
xcopy /E /I data\labels\抻面 data\labels_backup\抻面_备份_YYYYMMDD
```

或用项目中的 `scripts/backup_labels.py`（若支持抻面）。

### 步骤 2：用脚本创建新视频目录（不手改已有目录）

使用提供的 **`scripts/add_stretch_video_safe.py`** 创建新视频目录（如 cm13、cm14、cm15）：

- 只**新建** `data/labels/抻面/cm13/`（及可选 `data/processed/抻面/cm13/`）。
- 新目录中的 `classes.txt` **从主文件复制**，不自行编写、不用 labelImg 生成。
- **不会修改**主 `classes.txt`，也不会修改任何已有 cm1～cm12 下的任何文件。

这样从源头保证新视频的类别顺序与主文件一致。

### 步骤 3：抽帧并放入新视频目录

- 将新视频（如 2～3 个）抽帧得到的图片放到 `data/processed/抻面/cm13/`（及 cm14、cm15 若需要）。
- 确保图片与后续生成的标注文件名一一对应（同名 .jpg + .txt）。

### 步骤 4：在 labelImg 中标注时的硬性要求

- **类别顺序**：在 labelImg 中设置的类别列表必须与主文件**完全一致且顺序一致**：  
  第 1 类 = hand，第 2 类 = noodle_rope，第 3 类 = noodle_bundle。  
  不要新增、删除或调换顺序。
- **保存格式**：使用 YOLO 格式保存（每行：`class_id x_center y_center width height`，归一化 0～1）。
- **不要用 labelImg 修改 classes.txt**：  
  保存后若发现新视频目录下 `classes.txt` 被 labelImg 改过，**不要手动编辑**，用步骤 5 的脚本从主文件恢复。

### 步骤 5：标注完成后同步 classes.txt（防止被 labelImg 改坏）

标注完新视频后，运行 **`scripts/sync_stretch_classes_from_master.py`**：

- 用主文件 `data/labels/抻面/classes.txt` 的**当前内容**覆盖**所有** `data/labels/抻面/cm*/classes.txt`（包括 cm1～cm12 和新加的 cm13 等）。
- **只覆盖各视频目录下的 classes.txt**，不修改任何一张图片、不修改任何标注 `*.txt` 文件（框的 class_id 保持不变）。

这样即使 labelImg 改写了某个 cm* 的 classes.txt，也会被恢复成与主文件一致；只要你在 labelImg 里用的是「0=hand, 1=noodle_rope, 2=noodle_bundle」来标，标注文件里的 0/1/2 就仍然正确。

### 步骤 6：校验一致性后再训练

运行已有脚本：

```bash
python scripts/verify_classes_consistency.py
```

确认输出为「所有视频目录的 classes.txt 都与主文件一致」后再运行训练（如 `train_detection_model.py`），并在训练参数中把新视频（如 cm13、cm14、cm15）加入 `--videos` 列表。

## 四、要点小结

| 项目 | 要求 |
|------|------|
| 主 classes.txt | **不要用 labelImg 或手工修改**；唯一权威是当前 `data/labels/抻面/classes.txt`。 |
| 已有 cm1～cm7 | **不删除、不覆盖、不修改**；标注 cm10～cm12 后只用 `sync_stretch_classes_to_cm10_cm11_cm12_only.py`，绝不写入 cm1～cm7。 |
| 新视频目录 | 只通过 `add_stretch_video_safe.py` 创建，classes.txt 从主文件复制。 |
| labelImg 类别顺序 | 必须为：hand → noodle_rope → noodle_bundle（与主文件一致）。 |
| 标注 cm10～cm12 完成后 | 运行 `sync_stretch_classes_to_cm10_cm11_cm12_only.py`，再运行 `verify_classes_consistency.py`。 |

按上述流程操作，可以安全地多标注 2～3 个抻面视频，而**不会影响之前视频对应的 classes.txt 内容以及已有标注的标签映射和种类**。

## 五、为 cm10 / cm11 / cm12 抽帧并标注（不影响 cm1～cm7）

1. **抽帧（仅处理上述三段视频）**  
   将 raw 中对应视频抽帧到 processed，并只对这三个视频的 labels 目录同步主 `classes.txt`，不读取、不修改 cm1～cm7 及任何已有 `.txt`：
   ```bash
   python scripts/extract_stretch_frames_cm10_cm11_cm12.py
   ```
   - 视频来源：`data/raw/抻面/cm10.mp4`（或 .MP4）、cm11、cm12。  
   - 输出图片：`data/processed/抻面/cm10/cm10_00001.jpg` 等，命名与 cm1～cm7 一致。  
   - 抽帧规则：默认 2 fps，可用 `--fps 1.5` 等调整；可加 `--dry-run` 仅查看不写文件。

2. **用 labelImg 标注**  
   - 打开 `data/processed/抻面/cm10/`（及 cm11、cm12），保存到 `data/labels/抻面/cm10/` 等。  
   - 类别顺序**必须**与主文件一致：第 1 类 hand，第 2 类 noodle_rope，第 3 类 noodle_bundle；不增删类别。

3. **标注完成后（仅修正 cm10～cm12，不碰 cm1～cm7）**  
   运行**仅针对 cm10/cm11/cm12** 的同步脚本，再校验一致性。这样不会改写 cm1～cm7 下任何文件：
   ```bash
   python scripts/sync_stretch_classes_to_cm10_cm11_cm12_only.py
   python scripts/verify_classes_consistency.py
   ```
   确认输出为「所有视频目录的 classes.txt 都与主文件一致」后再参与训练。

## 六、命令汇总与数据保护承诺

### 推荐执行顺序（复制即用）

在项目根目录下执行：

```bash
# 1. 抽帧（仅生成 cm10、cm11、cm12 的图片，不碰 cm1～cm7）
python scripts/extract_stretch_frames_cm10_cm11_cm12.py

# 2. 用 labelImg 标注：打开 data/processed/抻面/cm10、cm11、cm12，保存到 data/labels/抻面/cm10、cm11、cm12
#    类别顺序必须为：hand → noodle_rope → noodle_bundle

# 3. 标注完成后：仅把主 classes.txt 同步到 cm10/cm11/cm12（不修改 cm1～cm7）
python scripts/sync_stretch_classes_to_cm10_cm11_cm12_only.py

# 4. 校验所有 cm* 的 classes.txt 与主文件一致
python scripts/verify_classes_consistency.py
```

可选：抽帧前先试跑，不写文件：

```bash
python scripts/extract_stretch_frames_cm10_cm11_cm12.py --dry-run
```

### 数据保护承诺

- **抽帧脚本** `extract_stretch_frames_cm10_cm11_cm12.py`：只向 `data/processed/抻面/cm10、cm11、cm12` 写入图片，只向 `data/labels/抻面/cm10、cm11、cm12` 写入/覆盖 `classes.txt`，**不读取、不修改** cm1～cm7、cm8、cm9 下任何文件。
- **同步脚本** `sync_stretch_classes_to_cm10_cm11_cm12_only.py`：仅用主 `classes.txt` 覆盖 **cm10、cm11、cm12** 下的 `classes.txt`，**绝不写入** cm1～cm9 或任何其他目录。
- **主 classes.txt**（`data/labels/抻面/classes.txt`）：请勿用 labelImg 另存或手工编辑；内容固定为三行：`hand`、`noodle_rope`、`noodle_bundle`。只要主文件不变，cm1～cm7 的标注语义不会因你标注 cm10～cm12 而改变。
- **labelImg 保存路径**：保存时务必选择 `data/labels/抻面/cm10`（或 cm11、cm12），不要选到 cm1～cm7 或主目录，避免误覆盖已有标注。

按上述命令与约定操作，**之前数据（cm1～cm7）的可靠性与稳定性不会因标注 cm10～cm12 而受到任何影响**。
