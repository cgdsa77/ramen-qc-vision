# 抻面模型升级建议

## 当前状态

- ✅ **已标注**: cm1~cm7（887张图片）
- ⏳ **待标注**: cm8, cm9, cm10, cm11, cm12（5个视频）

## 是否继续标注剩余视频？

### 推荐策略：**分阶段验证**

#### 阶段1：验证当前模型效果（推荐先做）

使用当前模型（mAP50=0.6940）在实际场景中测试：
- 测试不同场景的视频
- 检查识别准确率和误检情况
- 评估是否满足实际需求

**如果效果已经很好**：
- ✅ 可以跳过剩余标注，直接开始下面/捞面部分
- 节省时间，快速推进项目

**如果发现某些场景识别不佳**：
- 继续标注cm8~cm12，增加数据多样性
- 重新训练，预期mAP50可能提升到0.75~0.80

#### 阶段2：如果决定继续标注

**选项A：快速标注2-3个视频**（推荐）
- 标注cm8, cm9, cm10（约200-300张图片）
- 快速增加数据量，重新训练看效果
- 如果提升不明显，停止标注

**选项B：完整标注所有视频**
- 如果追求最高精度，标注所有剩余视频
- 预期mAP50可能提升到0.75~0.80
- 但需要投入更多时间

## 继续训练的方法

### 方法1：从当前最佳模型继续训练（推荐）

当前最佳模型已保存为：
- `models/stretch_detection/weights/best.pt`（Epoch 32, mAP50=0.6940）

**重要**：在训练新模型之前，先备份当前最佳模型！

```bash
# 1. 备份当前最佳模型
cp models/stretch_detection/weights/best.pt models/stretch_detection/weights/best_epoch32_backup.pt

# 2. 使用新数据集训练（包含cm8~cm12）
python src/training/train_detection_model.py \
    --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 cm8 cm9 cm10 \
    --epochs 200 \
    --batch 8 \
    --patience 30 \
    --no-resume  # 从头开始，或使用 --resume 从检查点继续
```

### 方法2：从检查点继续训练

如果之前训练中断，可以使用：
```bash
python src/training/train_detection_model.py \
    --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 cm8 cm9 cm10 \
    --epochs 200 \
    --batch 8 \
    --patience 30 \
    --resume  # 从 last.pt 继续训练
```

## 模型版本管理建议

建议保留不同版本的模型：

```
models/
├── stretch_detection/
│   └── weights/
│       ├── best.pt                    (当前使用的最佳模型)
│       ├── best_epoch32_backup.pt    (备份：Epoch 32, mAP50=0.6940)
│       └── best_epochXX_backup.pt    (新训练的备份)
```

这样可以：
- 随时回退到之前的模型版本
- 对比不同模型的效果
- 保留最佳性能的模型

## 总结建议

1. **先验证当前模型效果**，再决定是否继续标注
2. 如果继续标注，建议先标注2-3个视频测试效果
3. **始终备份当前最佳模型**，再进行新训练
4. 如果当前模型已经满足需求，可以跳过剩余标注，直接开始下面/捞面部分
