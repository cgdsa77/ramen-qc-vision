## 下面及捞面部分（整合）

### 类别定义

由于下面和捞面动作通常在一起，合并为一个部分，使用统一的4个类别：

1. **noodle_rope** - 面条
   - 标注正在捞出、已捞出在漏勺中、或其他情况下的面条
   - **重要**：在汤锅中的面条应标注为`soup_noodle`，不要标注为`noodle_rope`
   - 标注完整的面条区域

2. **hand** - 手部
   - 检测操作者的手部
   - 包括搅拌、检查、捞面等所有操作中的手部
   - 标注整个手部区域（包括手腕）

3. **tools_noodle** - 工具
   - 检测用于捞面的工具（漏勺、筷子等）
   - 标注整个工具的可见部分

4. **soup_noodle** - 汤中面条
   - 标注在汤锅中的面条
   - 用于区分汤中的面条和其他情况下的面条

### 标注注意事项

- 重点标注：手部 + 工具 + 面条的组合关系
- **面条分类**：
  - 在汤锅中的面条 → 标注为 `soup_noodle`
  - 正在捞出、已捞出等 → 标注为 `noodle_rope`
- 工具（漏勺等）是重要检测目标，需要准确标注
- 注意手部、工具、面条之间的空间关系
- **不再标注锅（pot）类别**

## 训练模型

标注完成后，使用相同的训练流程：

```bash
# 训练下面检测模型
python src/training/train_detection_model.py \
    --videos video1 video2 video3 ... \
    --epochs 200 \
    --batch 8 \
    --patience 30 \
    --no-resume

# 训练捞面检测模型
python src/training/train_detection_model.py \
    --videos video1 video2 video3 ... \
    --epochs 200 \
    --batch 8 \
    --patience 30 \
    --no-resume
```

**注意**：需要修改训练脚本中的路径，或者为每个阶段创建独立的训练脚本。

## 类别一致性检查

在开始标注前和标注过程中，定期运行一致性检查：

```bash
python scripts/verify_classes_consistency.py
```

确保所有视频子目录下的 `classes.txt` 与主文件一致。
