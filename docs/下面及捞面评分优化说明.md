# 下面及捞面评分优化说明

当前「下面及捞面」上传视频评分使用**占位规则**（覆盖率+密度 0~1 再换算为 5 分制），与抻面使用的「规则+图像特征」的专用评分不一致。若要像抻面一样做专用规则或训练评分模型后再接入，可按以下步骤优化。

---

## 一、现状对比

| 项目 | 抻面 (stretch) | 下面及捞面 (boiling_scooping) |
|------|----------------|-------------------------------|
| 评分器 | `StretchScorer`（`src/scoring/stretch_scorer.py`） | 无专用帧级评分器，仅占位逻辑 |
| 规则文件 | `data/scores/抻面/scoring_rules.json`（各属性 excellent/good/fair/poor 阈值） | 无 |
| 特征 | 检测框 + `ImageFeatureExtractor` 提取图像特征 | 仅检测框覆盖率、密度 |
| 输入 | 视频检测结果 + 视频路径（可读帧） | 同左，但只做占位统计 |
| 输出 | 1~5 分，`average_overall_score`、`class_average_scores` | 占位 0~1 换算为 1~5 分 |

项目内已有 **下面及捞面** 的标注与综合评分基础设施，但用于**已预处理、有标注的命名视频**（如 xl7），不能直接用于「任意上传视频」：

- 标注：`data/scores/下面及捞面/xl4/`、`xl5/`、`xl6/`、`xl7/`、`xl8/` 下的 `*_scores.json`（手部 position/action/angle/coordination/tool_coordination，面条 thickness/elasticity/ripeness 等）
- 骨架：`data/scores/下面及捞面/hand_keypoints/hand_keypoints_xl*.json`
- 综合评分器：`BoilingComprehensiveScorer`（`src/scoring/boiling_comprehensive_scorer.py`），按视频名加载上述数据做融合评分，**不接收「上传视频+检测结果」**

因此：上传视频的评分要优化，需要新增「仅凭检测结果（+ 可选图像特征）」的**帧级评分逻辑**，并接入接口。

---

## 二、优化路径一：专用规则（类似抻面）

目标：不做训练，用「规则 + 可选图像特征」对每帧检测结果打 1~5 分，再汇总成视频总分，与抻面流程一致。

### 1. 生成下面及捞面评分规则文件

- **数据来源**：现有 `data/scores/下面及捞面/xl*/*_scores.json` 中各属性的 1~5 分。
- **做法**：写脚本遍历这些 JSON，按属性统计 mean、std、分位数（如 poor=25% 分位、fair/good/excellent 等），输出 `data/scores/下面及捞面/scoring_rules.json`，格式可参考 `data/scores/抻面/scoring_rules.json`（含 `thresholds`、`overall_weights` 等）。
- **属性建议**（与现有标注一致）：
  - **hand**：position, action, angle, coordination, tool_coordination
  - **noodle_rope**：thickness, elasticity, integrity, ripeness, soup_adhesion, noodle_soup_ratio, distribution_state
  - **tools_noodle**：可先映射为 operation_standardization / tool_coordination 或单独设简单规则
  - **soup_noodle**：可复用 ripeness, distribution_state 或单独设「熟度/形态」相关阈值

### 2. 实现「下面及捞面」帧级评分器

- **新建**：`src/scoring/boiling_scooping_scorer.py`（或类似命名），参考 `StretchScorer` 的接口：
  - `score_frame(detections, image=None)`：对单帧检测列表（+ 可选原图）按规则打分，返回各类别分及 overall。
  - `score_video(video_detections, video_path=None)`：逐帧调用 `score_frame`，再汇总 `average_overall_score`、`class_average_scores`（1~5 分）。
- **输入**：`video_detections` 与抻面相同格式（每帧 `detections` 含 class、conf、xyxy 等）；`video_path` 可选，用于读帧做图像特征。
- **首版可简化**：仅用检测框的 **置信度 + 框大小/数量** 等简单特征，按规则映射到 1~5（例如高 conf 高覆盖→高分），不接图像特征也可上线。
- **进阶**：接入 `ImageFeatureExtractor` 或针对下面及捞面新写特征（如汤面区域、工具使用区域），再与 `scoring_rules.json` 中阈值结合，得到更细维度分。

### 3. 接入上传视频评分接口

- 在 `start_web.py` 的 `score_video` 中，当 `stage == "boiling_scooping"` 时：
  - 在「内容有效性校验」通过后，调用上述新评分器的 `score_video(video_detections, video_path=tmp_path)`。
  - 用其返回的 `average_overall_score`、`class_average_scores` 作为 `total_score`、`scores`，并设置 `rules_used` 为「下面及捞面专用规则」等说明。
- 若评分器加载失败或未实现，再回退到当前占位逻辑（0~1 换算 5 分制）。

---

## 三、优化路径二：训练评分模型

目标：用已有标注数据训练模型，对「每帧或整段视频」预测 1~5 分，再接入接口。

### 1. 标注与标签

- **标签**：从 `data/scores/下面及捞面/xl*/*_scores.json` 中按帧或按视频聚合得到 1~5 分（如每帧对 hand/noodle_rope/tools_noodle/soup_noodle 求平均或加权，或只做整体分）。
- **输入特征**（任选或组合）：
  - 检测结果：每帧各类别数量、平均置信度、覆盖率、框面积比等；
  - 骨架：若该视频已有关键点，可抽手部角度、协调性等统计量；
  - 图像：从视频帧裁剪检测框或整帧，用 CNN 等提特征。

### 2. 模型形式

- **回归**：预测每帧或整段视频的连续 1~5 分，再四舍五入或分段。
- **分类**：预测「差/一般/良/优」等档位，再映射到 1~5。
- 实现可放在 `src/scoring/` 下（如 `boiling_score_model.py`），训练脚本放在 `scripts/`（如 `train_boiling_scorer.py`），权重存 `data/scores/下面及捞面/` 或 `weights/`。

### 3. 接入方式

- 在「下面及捞面」帧级评分器中：若存在训练好的模型，则对每帧（或整段）提取特征 → 模型推理 → 得到 1~5 分；若无模型或推理失败，则回退到「专用规则」或占位逻辑。
- 接口仍返回 `total_score`、`scores`、`rules_used`（可注明「规则+模型」或「仅模型」）。

---

## 四、实施顺序建议

1. **短期（专用规则）**
   - 从 `xl*` 的 `*_scores.json` 统计生成 `data/scores/下面及捞面/scoring_rules.json`。
   - 实现 `BoilingScoopingScorer`（仅用检测置信度 + 简单几何统计 + 规则），在 `start_web.py` 中替代占位逻辑。
   - 不依赖骨架、不依赖图像特征即可上线，与抻面「5 分制」一致。

2. **中期（规则 + 图像特征）**
   - 复用或扩展 `ImageFeatureExtractor`，对下面及捞面四类目标提取特征，用 `scoring_rules.json` 的阈值做校准与打分。
   - 与抻面流程对齐，可解释性高。

3. **长期（模型融合）**
   - 用标注数据训练回归/分类模型，以检测+骨架+图像特征为输入，输出 1~5 分。
   - 与规则融合（如规则为主、模型微调）或逐步替代规则。
