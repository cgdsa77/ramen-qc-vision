# 手部姿态预处理方案使用说明

## 问题解决

### 原问题
1. ❌ 大部分存在手部骨架线缺失的状态
2. ❌ 骨架线出现显示一段时间后，由于手部动作已经实时变化，而原先显示的骨架线仍在那一帧的位置，所以看起来会有错位的感觉
3. ❌ 骨架线与视频动作实时显示，但是这个并未达到效果

### 解决方案：预处理方案 ✅

**核心思路**：先批量处理视频，生成所有帧的关键点数据（JSON格式），前端直接读取JSON，实现无延迟的实时显示。

**优势**：
- ✅ **无延迟**：前端直接读取JSON，没有API调用延迟
- ✅ **完美同步**：骨架线与视频播放完全同步，无错位
- ✅ **可提高精度**：可以使用OpenPose等更高精度的工具进行预处理
- ✅ **性能稳定**：不依赖实时处理速度

## 使用步骤

### 1. 预处理视频（生成关键点数据）

运行预处理脚本，批量处理所有视频：

```bash
python scripts/extract_hand_keypoints_from_video.py
```

**处理内容**：
- 自动扫描 `data/raw/抻面/` 和 `data/raw/下面及捞面/` 目录下的所有视频
- 对每个视频逐帧提取手部关键点
- 生成JSON文件保存到：
  - `data/scores/抻面/hand_keypoints/hand_keypoints_{视频名}.json`
  - `data/scores/下面及捞面/hand_keypoints/hand_keypoints_{视频名}.json`

**输出格式**：
```json
{
  "video": "cm1",
  "fps": 30.0,
  "total_frames": 1500,
  "width": 1920,
  "height": 1080,
  "detected_frames": 1200,
  "missing_frames": 300,
  "frames": [
    {
      "frame_index": 0,
      "hands": [
        {
          "id": 0,
          "keypoints": [
            {"x": 100, "y": 200, "z": 0, "confidence": 0.9},
            ...
          ]
        }
      ]
    },
    ...
  ]
}
```

### 2. 访问Web界面

1. 启动Web服务器（如果还没启动）：
   ```bash
   python start_web.py
   ```

2. 访问手部姿态实时展示页面：
   ```
   http://127.0.0.1:8000/web/hand_pose_realtime.html
   ```

3. 选择视频，点击播放
   - 系统会自动加载对应的JSON文件
   - 骨架线会与视频完美同步显示

## 技术实现

### 前端改进

**之前（实时API调用）**：
- 每帧都调用 `/api/process_frame_for_pose`
- 有网络延迟和处理延迟
- 导致错位和不流畅

**现在（预处理JSON）**：
- 一次性加载JSON文件
- 根据视频播放时间直接查找对应帧数据
- 无延迟，完美同步

### 后端API

**API端点**：`GET /api/hand_keypoints/{video_name}?stage=stretch`

**参数**：
- `video_name`: 视频名称（如 `cm1`）
- `stage`: 阶段类型
  - `stretch`: 抻面（默认）
  - `boiling_scooping`: 下面及捞面

**返回**：
```json
{
  "success": true,
  "data": {
    "video": "cm1",
    "fps": 30.0,
    "total_frames": 1500,
    "frames": [...]
  }
}
```

## 处理所有视频

### 抻面视频

预处理脚本会自动处理 `data/raw/抻面/` 目录下的所有 `.mp4` 文件。

### 下面及捞面视频

预处理脚本会自动处理 `data/raw/下面及捞面/` 目录下的所有 `.mp4` 文件。

## 性能说明

### 预处理时间

- **MediaPipe**：约 1-2 分钟/视频（30 FPS，1500帧）
- **OpenPose**：约 5-10 分钟/视频（更精确但更慢）

### 文件大小

- 每个视频的JSON文件约 5-20 MB（取决于视频长度和检测到的帧数）

### 前端性能

- JSON加载：< 1秒
- 骨架线绘制：实时，无延迟
- 内存占用：约 20-50 MB（取决于视频长度）

## 提高检测精度

如果MediaPipe预处理后仍有大量缺失，可以考虑：

### 方案1：使用OpenPose预处理（推荐）

1. 安装OpenPose（使用Docker或Python 3.7环境）
2. 修改 `extract_hand_keypoints_from_video.py`，使用OpenPose替代MediaPipe
3. 重新运行预处理

### 方案2：优化MediaPipe参数

在 `extract_hand_keypoints_from_video.py` 中可以调整：
- 图像预处理（缩放、增强）
- 检测阈值
- 多尺度检测

## 常见问题

### Q1: 预处理脚本运行很慢？

**A**: 这是正常的。预处理需要逐帧处理视频，MediaPipe约1-2分钟/视频。可以：
- 使用GPU加速（如果支持）
- 使用OpenPose（虽然更慢但精度更高）
- 分批处理视频

### Q2: 某些视频检测不到手部？

**A**: 可能原因：
1. 视频中手部被遮挡或太小
2. MediaPipe精度限制
3. 光照条件不好

**解决方案**：
- 使用OpenPose（精度更高）
- 检查视频质量
- 调整检测参数

### Q3: JSON文件很大？

**A**: 这是正常的。每个视频的JSON包含所有帧的关键点数据。可以：
- 只保存检测到手的帧（减少文件大小）
- 使用压缩（gzip）
- 定期清理不需要的JSON文件

### Q4: 如何更新预处理数据？

**A**: 直接重新运行预处理脚本，会覆盖旧的JSON文件。
