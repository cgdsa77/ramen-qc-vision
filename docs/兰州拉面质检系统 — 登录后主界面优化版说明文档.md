# 兰州拉面质检系统 — 登录后主界面优化版说明文档

本文档基于原界面结构，仅针对**登录后主界面**进行优化，聚焦**视觉质感强化、Popmotion动效精细化、响应式/无障碍体验提升**，不涉及登录页修改，不引入本地真实图片。Cursor可直接按此文档修改代码，保留原有功能逻辑。

---

## 一、项目与系统简介

- **系统名称**：面向兰州拉面制作的标准化流程视觉质检系统  

- **主入口**：单页应用，登录后根据角色展示不同主题与文案的主 dashboard  

- **主要功能**：抻面/下面及捞面的流程检测、标注工具、综合评分、姿态视频展示；管理员另有用户管理与系统设置，培训师另有学员管理与评分标准配置  

- **优化核心目标**：通过角色主题配色细化、动效自然化、交互反馈增强，提升界面专业度与使用体验，同时保证性能与无障碍。

---

## 二、技术栈与文件位置

- **主界面文件**：`web/index.html`（HTML + 内联 CSS + 内联 JS，无前端框架）  

- **字体**：Google Fonts — Noto Serif SC、Noto Sans SC；优化行高（标题1.3、正文1.6）、字间距（标题0.02em、正文0.01em）  

- **动效库**：Popmotion（已集成），通过 `window.popmotion` 或 `popmotion.bundle.js` 引入；优化动画曲线、弹簧参数、新增区块/弹窗动效  

- **后端**：FastAPI（`start_web.py`），会话与权限通过 `/api/auth/session`、`api/auth/login` 等接口维护

- **角色码**：`0` = 管理员，`1` = 培训师，`2` = 厨师/学员

    - 导航按钮的显隐（`data-role-visible`）；

    - 副标题、区块标题、卡片标题与描述（`applyRoleCopy(role)` 从 `ROLE_COPY` 写入，文案优化为角色化表述）；  

    - 整页配色、背景、动效参数（见第五、九章）；  

- **优化补充**：角色差异化字体权重（管理员标题600、培训师500、学员400），强化角色感知。

## 九、动效说明（Popmotion·优化版）

### 9.1 CSS 动效

- **顶条入场**：`body.main-page .top-bar` 的 `topBarReveal` 改用 Popmotion `spring` 曲线（`stiffness: 120, damping: 15`），scaleX 0→1；

- **按钮 hover**：所有.btn/.btn-primary 增加渐变偏移（`background-position: 100% 0`）+ 1px内发光。

### 9.2 Popmotion 核心动效（runPageAnimations）

- **Hero 区域**：

    - 主标题：逐字动画（opacity 0→1 + translateY 0→-5px + 旋转0.5deg）；

    - 副标题：淡入延迟0.8s，呼吸感增强；

- **卡片动效**：

    - 入场：scale(0.98→1) + opacity(0→1)，stagger延迟0.08s；

    - 悬停：角色差异化上浮（role-0: -8px、role-1: -12px、role-2: -10px）+ 阴影放大 + 旋转0.8deg，弹簧参数 `stiffness: 100, damping: 18`；

- **区块入场**：IntersectionObserver触发，.section 实现 opacity(0→1) + translateY(15px→0)，子元素stagger 0.08s；

- **弹窗动效**：所有.modal-box 打开（scale0.95→1 + opacity0→1 + translateY20px→0）、关闭（scale1→0.98 + opacity1→0 + translateY0→10px）。

### 9.3 性能优化

- 非首屏动效延迟触发（IntersectionObserver），避免同时运行过多Popmotion实例；

- 卡片/按钮动效仅在hover时触发，且弹簧参数统一，减少性能消耗。
