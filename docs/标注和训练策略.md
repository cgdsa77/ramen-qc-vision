# 标注和训练策略建议

## 当前情况

- ✅ 已标注：cm1, cm2, cm3（3个视频）
- ⏳ 待标注：cm4, cm5, ..., cm12（9个视频）
- ❌ 检测准确度：很低，基本识别错误

## 问题分析

**3个视频的数据量太少**，对于训练YOLO检测模型来说：
- 数据多样性不足
- 模型容易过拟合
- 泛化能力差

## 推荐策略：分批标注和训练

### 方案一：快速迭代（推荐）

**第一阶段：标注到5-6个视频**
1. 继续标注 cm4, cm5（或 cm4, cm5, cm6）
2. 重新训练模型
3. 测试检测效果

**第二阶段：根据效果决定**
- 如果效果有明显提升：继续标注 cm6, cm7, cm8
- 如果效果仍不理想：检查标注质量，调整训练参数

**第三阶段：完善数据集**
- 标注剩余视频
- 最终训练完整模型

### 方案二：一次性标注（不推荐）

- 优点：数据完整
- 缺点：如果标注有问题，需要全部重来
- 风险：标注9个视频后发现模型仍有问题

## 具体建议

### 1. 先标注2-3个视频（cm4, cm5, cm6）

**原因：**
- 从3个增加到5-6个，数据量翻倍
- 可以快速验证标注质量和训练效果
- 如果效果好，继续标注；如果不好，及时调整

### 2. 重新训练模型

标注完成后运行：
```bash
python src/training/train_detection_model.py
```

**注意：** 需要修改训练脚本，将新标注的视频加入训练集。

### 3. 测试效果

- 上传未标注的视频（如 cm7）进行测试
- 查看检测准确度
- 评估是否需要更多数据

### 4. 迭代优化

根据测试结果：
- **效果好**：继续标注剩余视频
- **效果一般**：检查标注质量，可能需要调整训练参数
- **效果差**：检查标注是否正确，类别是否一致

## 训练参数优化建议

如果数据量增加到5-6个视频后，可以尝试：

1. **增加训练轮数**
   ```python
   epochs=100  # 从50增加到100
   ```

2. **调整批次大小**
   ```python
   batch=8  # 如果显存不足，可以减小
   ```

3. **数据增强**
   - YOLO默认会做数据增强
   - 可以检查是否有足够的多样性

## 标注质量检查清单

在继续标注前，请检查：

- [ ] 标注框是否准确（完全包含目标）
- [ ] 类别是否正确（hand, noodle_rope, noodle_bundle, pot_or_table）
- [ ] 是否有遗漏的目标
- [ ] 不同视频的标注标准是否一致

## 快速开始

### 步骤1：标注 cm4, cm5

使用 labelImg 继续标注：
- 打开 `data/processed/抻面/cm4/` 和 `cm5/`
- 标注所有图片
- 确保标注文件保存在 `data/labels/抻面/cm4/` 和 `cm5/`

### 步骤2：更新训练脚本

修改 `src/training/train_detection_model.py`：
```python
standard_videos = ['cm1', 'cm2', 'cm3', 'cm4', 'cm5']  # 添加新视频
```

### 步骤3：重新训练

```bash
python src/training/train_detection_model.py
```

### 步骤4：测试效果

- 重启服务器
- 上传 cm6 或 cm7 视频测试
- 查看检测准确度

## 预期效果

- **5-6个视频**：检测准确度应该有明显提升（预计60-70%）
- **8-10个视频**：检测准确度应该达到较好水平（预计80%+）
- **12个视频**：检测准确度应该达到良好水平（预计85%+）

## 总结

**建议：先标注2-3个视频（cm4, cm5, cm6），重新训练测试效果，再决定是否继续标注更多视频。**

这样可以在标注工作量和质量之间找到平衡点。
