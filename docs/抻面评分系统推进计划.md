# 抻面评分系统推进计划

## 📊 当前状态总结

### ✅ 已完成的工作

1. **标准数据集建立**
   - 在 `cm1`、`cm2`、`cm3` 三个视频中各选取了约10个关键帧
   - 对关键帧进行了手动评分，包含：
     - `noodle_rope`（面条）：厚度、弹性、光泽、完整性
     - `hand`（手部）：位置、动作、角度、协调性
     - `noodle_bundle`（面束）：紧实度、均匀度
   - 所有评分数据保存在 `data/scores/抻面/standard_dataset.json`

2. **评分规则建立**
   - 基于标准数据集自动生成了评分规则和阈值（`scoring_rules.json`）
   - 包含各类别属性的阈值（优秀、良好、一般、较差）
   - 定义了各类别和属性的权重配置

3. **基础评分器实现**
   - 实现了 `StretchScorer` 类（`src/scoring/stretch_scorer.py`）
   - 集成到 Web API 中（`/api/score_video`）
   - 可以返回JSON格式的评分结果

### ⚠️ 当前问题

1. **评分逻辑过于简化**
   - 当前仅基于检测置信度计算分数（`base_score = conf * 5`）
   - **缺少真正的图像特征分析**：
     - 面条粗细无法从检测框大小准确判断
     - 光泽度、弹性等属性未提取
     - 手部姿态角度未计算
     - 面束紧实度和均匀度未分析

2. **缺少详细报告**
   - 仅返回JSON数据，缺少可视化报告
   - 无法直观看到问题标注和改进建议

3. **缺少验证机制**
   - 未评估自动评分与手动评分的一致性
   - 无法量化评分系统的准确性

## 🎯 下一步推进计划

### 阶段一：改进评分逻辑（核心任务）

#### 1.1 实现图像特征提取模块

**目标**：从检测框区域提取真实的图像特征，而非仅依赖置信度

**需要提取的特征**：

**A. `noodle_rope`（面条）特征：**
- **厚度（thickness）**：
  - 计算检测框区域内面条的平均宽度（边缘检测 + 轮廓分析）
  - 与标准厚度范围对比
- **光泽度（gloss）**：
  - 计算区域的亮度方差和对比度
  - 高光泽 = 高亮度 + 低方差
- **完整性（integrity）**：
  - 检测是否有断裂（边缘连续性分析）
  - 检测是否有粘连（轮廓数量分析）
- **弹性（elasticity）**：
  - 基于序列帧分析形变程度（需要多帧上下文）
  - 暂时使用检测框稳定性作为代理

**B. `hand`（手部）特征：**
- **位置（position）**：
  - 检测框中心位置相对于画面和面条的位置
  - 判断是否在合理操作区域内
- **角度（angle）**：
  - 计算检测框的旋转角度（主轴方向）
  - 或通过边缘检测计算手部指向角度
- **动作（action）**：
  - 基于序列帧分析手部运动轨迹
  - 判断动作是否符合标准（需要姿态估计，暂缓）
- **协调性（coordination）**：
  - 分析双手（如果有多个hand检测框）的对称性和配合度

**C. `noodle_bundle`（面束）特征：**
- **紧实度（tightness）**：
  - 计算面束区域内的密度（像素密度、边缘密度）
  - 紧实 = 高密度 + 清晰轮廓
- **均匀度（uniformity）**：
  - 计算面束内各部分的粗细一致性
  - 使用局部方差和标准差

**实施步骤**：
1. 创建 `src/features/image_feature_extractor.py`
2. 实现各类别的特征提取函数
3. 将特征值映射到1-5分评分区间
4. 更新 `StretchScorer.score_detection()` 方法

#### 1.2 集成特征提取到评分器

**修改 `src/scoring/stretch_scorer.py`**：
- 在 `score_detection()` 中调用特征提取模块
- 需要传入原始图像和检测框坐标
- 基于提取的特征值（而非置信度）进行评分

**API修改**：
- 修改 `/api/score_video`，确保传递原始图像数据给评分器

### 阶段三：验证和优化评分系统

#### 3.1 评分一致性验证

**目标**：评估自动评分与手动评分的一致性

**方法**：
1. 使用已有的手动评分数据（`standard_dataset.json`）
2. 对相同帧进行自动评分
3. 计算一致性指标：
   - 平均绝对误差（MAE）
   - 相关系数（Pearson/Spearman）
   - 分类准确率（优秀/良好/一般/较差）

**实施步骤**：
1. 创建 `scripts/evaluate_scoring_consistency.py`
2. 输出验证报告，包含：
   - 各属性的一致性统计
   - 不一致案例分析
   - 改进建议

#### 3.2 阈值和权重优化

**目标**：基于验证结果调整评分规则

**方法**：
1. 如果发现系统性偏差，调整阈值
2. 如果发现某些属性权重不合理，调整权重
3. 迭代优化直到达到可接受的一致性水平

**实施步骤**：
1. 创建 `scripts/optimize_scoring_rules.py`
2. 基于验证结果自动或半自动调整规则
3. 重新生成 `scoring_rules.json`

## 🛠️ 技术实现要点

### 图像特征提取技术栈

- **OpenCV**：图像处理和边缘检测
- **NumPy**：数值计算
- **scikit-image**：高级图像分析（可选）

### 特征提取算法建议

1. **厚度计算**：
   ```python
   - 在检测框区域内提取ROI
   - 使用Canny边缘检测
   - 计算垂直于面条方向的平均宽度
   ```

2. **光泽度计算**：
   ```python
   - 计算ROI的亮度均值（转换为HSV，取V通道）
   - 计算亮度的标准差（高光泽 = 高均值 + 低标准差）
   ```

3. **角度计算**：
   ```python
   - 使用PCA计算检测框的主轴方向
   - 或使用Hough变换检测直线方向
   ```

4. **紧实度计算**：
   ```python
   - 计算ROI内的边缘密度（Canny边缘像素占比）
   - 计算轮廓的紧凑度（面积/周长²）
   ```
