## 二、阈值（thresholds）体系 全维度严谨化优化

阈值是评分体系的核心基准，本次优化从「经验固定值」升级为「数据驱动+行业标准+场景适配」的科学体系，完全复用原属性标签与四级映射逻辑。

### 2.1 阈值统计逻辑的科学重构

#### 现存痛点

原阈值仅参考约数`mean/std`，无严谨的统计与行业依据；部分阈值存在无区分度瑕疵（如下面及捞面`position`的`excellent=good=4`）；阈值划分无统一标准，人工调整随机性强。

#### 优化方案

1. **统一四级阈值的统计与行业标准**

基于有效标注数据集，采用「正态分布+行业分位数」双维度校准，明确四级阈值的刚性定义，替代原有经验值：

|等级|分值|统计标准（双维度取高值）|行业对应水平|
|---|---|---|---|
|excellent|5分|≥标注值P90分位数，且≥mean+1.28*std|行业前10%优秀水平|
|good|4分|≥标注值P75分位数，且≥mean+0.67*std|行业前25%良好水平|
|fair|3分|≥标注值P50分位数（中位数），且≥mean-0.67*std|行业平均合格水平|
|poor|2分|≥标注值P25分位数，且≥mean-1.28*std|行业后25%不合格临界|
|＜poor阈值|1分|＜标注值P25分位数，且＜mean-1.28*std|严重不合格水平|
1. **瑕疵阈值专项修正与格式规范**

    - 针对原规则中无区分度的阈值（如`position`/`thickness`的`excellent=good=fair`），按上述标准重新计算，确保`excellent＞good＞fair＞poor`的严格单调性，消除无区分度区间

    - 所有阈值、`mean`、`std`统一保留2位小数，JSON文件数值格式标准化，避免四舍五入歧义

    - 新增阈值区间刚性约束：所有属性阈值严格锁定在[1,5]区间，杜绝阈值超出评分范围的情况

2. **配套工具升级**

重构`build_scoring_rules.py`/`build_boiling_scooping_scoring_rules.py`，内置上述分位数+正态分布的标准化计算逻辑，替代原有经验值生成逻辑，支持一键生成合规阈值。

#### 落地校验标准

所有属性阈值满足严格单调性；阈值计算可通过标注数据集100%复现；同属性不同批次标注数据的阈值波动≤0.2。

## 三、权重体系 精细化与可验证性优化

完整保留原规则的类别权重（`overall_weights`）、子权重（`weights`）核心结构，仅从「数据验证、场景适配、偏差补偿」三个维度优化，确保权重分配与实际业务关注点高度匹配。

### 3.1 权重合理性的量化验证与优化

#### 现存痛点

权重分配为固定经验值，无数据支撑，无法验证各属性对最终评分的实际贡献度，与行业培训师的评分习惯脱节。

#### 优化方案

1. **数据驱动的权重校准**

基于「多元线性回归+皮尔逊相关性分析」，验证并优化权重分配，确保权重与属性实际贡献度匹配：

- 以人工标注的最终评分为因变量，各属性原始值为自变量，做多元线性回归，得到各属性的实际贡献系数

- 计算各属性与对应类别总分的皮尔逊相关系数，相关系数越高，对应子权重越高

- 示例：原抻面`hand`类别下，`action`的相关系数为0.82，`position`为0.65，可将`action`子权重从0.3上调至0.35，`position`从0.25下调至0.2，不改变属性本身

1. **刚性约束与核心保障**

    - 权重和强制校验：所有类别内的子权重之和严格=1（允许±0.001浮点误差），`overall_weights`所有类别权重之和严格=1

    - 核心属性最低权重保障：针对行业核心关注的属性，设置最低权重阈值，如抻面`noodle_rope`的`integrity`、`hand`的`action`，子权重不得低于0.25，确保核心技能点的权重占比

2. **配套工具升级**

新增权重分析脚本，自动计算属性贡献系数与相关性，输出权重优化建议；校验脚本新增权重和校验、核心属性最低权重校验。

#### 落地校验标准

权重分配与属性贡献系数的匹配度≥90%；所有权重和校验100%通过；核心属性权重不低于最低阈值。

#### 落地校验标准

## 四、评分计算逻辑 鲁棒性与边界场景全补全优化

完整保留原规则「单检测框→单帧→视频」的核心计算流程与归一化公式，仅对不合理逻辑、边界场景、歧义点进行补全与优化，确保评分结果可复现、无失真。

### 4.1 偏置逻辑的动态化与合理化重构

#### 现存痛点

抻面固定给`hand+0.7`、`noodle_rope+0.8`的线性偏置，极端值（1分/5分）加偏置后无意义，过度修正导致评分失真，无校准机制。

#### 优化方案

1. **替换固定偏置为分段动态偏置**

仅对中间区间的分数做修正，避免极端值无效修正，重构`StretchScorer.calculate_weighted_score`中的偏置逻辑：

```Python

def dynamic_bias_adjust(score, bias, category):
    """
    动态偏置调整函数
    :param score: 原始加权分
    :param bias: 类别基准偏置（hand:0.7, noodle_rope:0.8）
    :param category: 类别名称
    :return: 调整后的分数，严格锁定[1,5]区间
    """
    # 仅对(1.2, 4.8)区间的分数做偏置调整，避免极端值失真
    if 1.2 < score < 4.8:
        # 偏置随分数动态调整：越接近均值，偏置越接近基准值；越接近两端，偏置越小
        mean_score = RULES["thresholds"][category]["mean"]
        std_score = RULES["thresholds"][category]["std"]
        distance_from_mean = abs(score - mean_score) / std_score
        dynamic_bias = bias * max(0, 1 - distance_from_mean / 3)
        score += dynamic_bias
    # 严格裁剪到[1,5]区间
    return min(max(score, 1), 5)
```

1. **偏置开关与校准机制**

    - 新增「偏置启用开关」，当自动评分与人工评分的整体偏差率＜5%时，可自动关闭偏置

    - 每季度校准一次基准偏置值，基于最新的人工-自动评分偏差，调整`hand/noodle_rope`的基准偏置，偏差为负则上调，偏差为正则下调

    - 下面及捞面同步新增可选动态偏置功能，默认关闭，可基于评分偏差按需启用

#### 落地校验标准

偏置调整后分数区间严格在[1,5]；极端值无无效修正；偏置调整后人工-自动评分偏差率降低≥30%。

### 4.2 边界场景的明确定义与计算逻辑补全

#### 现存痛点

「有检测到的类别」定义模糊、单帧无有效检测框、视频全程某类别无检测、低置信度检测框等边界场景无明确规则，导致评分结果不可复现。

#### 优化方案

1. **明确刚性判定标准，写入规则与代码**

    - 有效检测框：检测置信度≥0.3，且框体尺寸符合对应类别的物理范围（如`hand`框体画面占比0.5%-20%），避免误检

    - 单帧有效类别：该帧内某类别下有效检测框数量≥1，判定为该帧「出现类别」

    - 视频级有效类别：该类别在视频总帧数中，出现占比≥30%，判定为视频级「有效参与类别」，避免单帧误检导致权重归一化偏差

2. **无有效检测场景的兜底逻辑**

    - 单帧无任何有效检测框：该帧`overall_score`不参与视频总分计算，避免0分拉低整体评分

    - 视频全程某类别无有效检测：提供两种可配置模式，默认沿用原规则的「排除不参与」模式，可选「按0分参与加权」模式，由管理员在规则中配置

    - 单帧仅单个有效类别：该帧`overall_score=该类别加权分`，同时设置单类别最高权重上限0.8，剩余0.2权重按其他类别的历史均值分补充，避免单类别权重100%导致的评分失真

3. **低质量数据过滤逻辑**

    - 检测框置信度＜0.3的，不参与评分计算

    - 单帧画面模糊度＞阈值（清晰度评分＜30），该帧不参与评分计算

    - 视频有效帧数＜总帧数的50%，触发「视频质量过低」提示，不输出有效评分

#### 落地校验标准

所有边界场景均有明确计算逻辑；同一段视频多次评分结果完全一致；低质量数据过滤正常生效。

### 4.3 视频级评分的优化与归一化透明化

#### 现存痛点

视频总分仅用帧平均分，未考虑操作关键帧的权重；归一化过程不透明，无法追溯。

#### 优化方案

1. **新增关键帧加权机制**

保留原帧平均逻辑为默认模式，新增可选的关键帧加权模式：

- 基于动作识别，将抻面「出条」、下面「下锅」、捞面「起锅」等关键动作帧，权重设置为普通帧的2倍

- 非关键动作帧权重为1，最终视频总分=Σ(单帧overall_score * 帧权重) / Σ(帧权重)

1. **归一化逻辑透明化与可追溯**
