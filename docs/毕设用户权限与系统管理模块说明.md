# 毕设用户权限与系统管理模块（单web主页面适配版）- 适配cursor开发

核心目标：基于你现有「单个web主页面+多模块」的现状，最小侵入式融入角色登录、权限管控功能，新增必要子页面（弹窗/跳转页），不破坏原有模块逻辑，适配cursor读取文档后直接开发，实现多角色差异化页面展示，让系统从单demo升级为可落地业务系统，贴合AY类课题工程化要求。

核心原则：1. 不新增多余独立页面，仅新增「登录弹窗/简易登录页」「用户管理弹窗」2个核心子页面，其余功能均在原有主页面内通过“模块显示/隐藏”实现；2. 原有模块（视频采集、实时检测、评分报告、数据集标注等）保留，仅根据角色权限控制显示/隐藏，不修改原有模块核心代码；3. 方案条理清晰、细节明确，适配cursor读取后直接用于开发，无需额外梳理。

## 一、整体页面规划（贴合单web主页面，cursor可直接识别）

现有web主页面保持不变，新增2个子页面（以弹窗/简易跳转页形式，避免多页面复杂开发），整体结构如下，所有页面/模块均标注开发要点，适配cursor读取：

- 核心页面：原有web主页面（承载所有核心模块，根据角色权限动态显示/隐藏模块）

- 新增子页面1：登录弹窗（优先）/ 简易登录页（备选）- 打开系统先显示，登录后跳转至主页面，适配原有主页面风格

- 新增子页面2：用户管理弹窗 - 仅管理员可见，点击主页面“用户管理”按钮弹出，完成用户新增、编辑、删除等操作，无需新增独立页面

补充：所有新增页面/弹窗，均沿用原有web主页面的UI风格、颜色、字体，保证视觉统一，cursor读取后可直接匹配原有页面样式开发。

## 二、基础核心功能（必做，适配单页面，cursor友好）

1. **登录功能（核心，适配单页面，cursor可直接开发）**

    - 页面形式：优先做「登录弹窗」（嵌入原有主页面，打开系统自动弹出，登录成功后弹窗关闭，显示主页面）；若弹窗开发不便，可做「简易登录页」（单独1个简单页面，登录后跳转至原有主页面），二选一即可，推荐弹窗（不新增多余页面）。

    - 登录弹窗核心元素（cursor可直接参考开发）：账号输入框、密码输入框、登录按钮、“忘记密码”链接（仅显示，点击跳转至管理员重置密码入口）、关闭按钮（禁用，必须登录才能进入主页面），弹窗样式与原有主页面保持一致，简洁不复杂。

    - 数据库对接（同之前，cursor可直接读取字段开发）：MySQL用户表核心字段（id：主键自增、username：唯一登录账号、password：MD5加密、role：角色标识0=管理员/1=培训师/2=厨师、name：真实姓名、create_time：创建时间、status：0禁用/1正常）。

    - 核心逻辑（cursor可直接编写代码）：登录校验（账号密码错误提示“账号或密码错误”）、禁用账号提示（“账号已禁用，请联系管理员”）、登录状态保存（Session存储，有效期24小时）、登录后根据角色，动态显示主页面对应模块（关键，适配单页面核心需求）。

    - 简化开发：忘记密码功能简化为“联系管理员重置”，无需开发手机号/邮箱找回，cursor可快速实现。

2. **角色划分（固定3类，贴合单页面展示，与权限、模块联动）**与数据库role字段一一对应，cursor可直接关联权限与模块显示逻辑，无需额外调整：

    - 管理员（role=0）：主页面显示所有模块（原有模块+新增管理入口），可操作所有功能；

## 八、部署与需您手动操作说明（当前实现）

当前实现**无需安装 MySQL**，开箱即用：

- **数据库**：默认使用 **SQLite**，数据文件自动创建在 `data/ramen_auth.db`。您**不需要**下载或安装 MySQL，直接运行 `python start_web.py` 即可。
- **默认管理员账号**：首次运行后，使用账号 **admin**、密码 **admin123** 登录。登录后建议在「用户管理」中修改默认密码或新增其他管理员。
- **角色**：与文档一致，role=0 管理员、role=1 培训师、role=2 厨师/学员；登录后主页面按角色自动显示/隐藏对应模块与按钮。

若将来需要改为 **MySQL**，您只需完成以下步骤（建库建表与代码已就绪）：

1. **安装 MySQL 服务**：从 [MySQL 官网](https://dev.mysql.com/downloads/mysql/) 或系统包管理器安装 MySQL，并启动服务。
2. **安装 Python 驱动**：在项目目录执行  
   `pip install pymysql`
3. **建库、建表**：在命令行执行（将 `root` 和密码换成你的 MySQL 账号密码）。  
   - CMD / Bash：`mysql -u root -p你的密码 < scripts/init_mysql_auth.sql`  
   - **PowerShell**（Windows 下 `<` 无效，请用）：  
     `Get-Content .\scripts\init_mysql_auth.sql -Raw -Encoding UTF8 | & "C:\Program Files\MySQL\MySQL Server 9.6\bin\mysql.exe" -u root -p你的密码`  
   - 或在 MySQL 客户端中执行 `scripts/init_mysql_auth.sql`。脚本会创建数据库 `ramen_qc`、表 `users`，并在表为空时插入默认管理员 admin / admin123。
4. **配置连接**：复制 `configs/db_config.example.json` 为 `configs/db_config.json`，填写 MySQL 地址、端口、用户名、**密码**、数据库名（`ramen_qc`）。若已由他人或脚本生成好 `db_config.json`，只需核对其中的 `password` 是否与 MySQL root 密码一致。保存后重启 Web 服务即可自动使用 MySQL。

**总结**：不用 MySQL 时无需任何安装；使用 MySQL 时只需安装 MySQL 服务并执行上述 2～4 步，建库建表与代码已由项目提供。
