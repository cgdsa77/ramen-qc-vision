# 实时监测识别说明

本文说明实时监测页的检测逻辑、推荐外源模型与可选优化方式。

## 当前检测方式

- **手部（hand）**：推荐使用 **MediaPipe Hands**。页面勾选「手部」时由 MediaPipe 提供框。
- **头部（head）**：使用 **MediaPipe Face Detector**。页面勾选「头部」时显示人脸框，并用于抑制与头部重叠的 noodle 误检。需在 `weights/mediapipe/` 下放置 `face_detection_short_range.tflite`（可运行 `python scripts/download_face_detector_model.py` 自动下载）。
- **其他类别（noodle_rope、面条等）**：项目 YOLO。抻面阶段对 noodle_rope/noodle_bundle 提高置信度门槛（≥0.7）以减轻窗帘/褶皱误判。  
  - 抻面：`models/stretch_detection/weights/best.pt`  
  - 下面及捞面：`models/boiling_scooping_detection/weights/best.pt`

### 依赖（手部使用 MediaPipe 时）

- 安装：`pip install mediapipe`（或 `pip install -r requirements-realtime.txt`）。
- **MediaPipe 0.10+**：需在项目下放置手部模型文件 `weights/mediapipe/hand_landmarker.task`（可从 [MediaPipe 模型](https://ai.google.dev/edge/mediapipe/solutions/vision/hand_landmarker/index#models) 下载）。若未放置，将自动尝试旧版 API（需 `mediapipe<0.10`）。
- 若不安装 mediapipe，手部会回退为 YOLO 检测，功能仍可用。

## 已做的代码侧优化

1. **手部外源模型（MediaPipe Hands）**  
   - 手部由 Google MediaPipe 检测，负样本（背景、人脸）区分更好，手部召回与框稳定性更高。  
   - 页面默认开启「手部 MediaPipe」，可随时关闭对比 YOLO 效果。

2. **按类别固定框颜色**  
   - 手=橙、面条/ rope=绿等，同一类别每帧颜色一致，**不再红绿蓝闪烁**。

3. **置信度与 NMS**  
   - 实时流默认置信度 **0.4**；页面可选 0.3～0.6。  
   - YOLO 使用 iou=0.45；同类别再做 NMS，减轻多框重叠。

## 若仍不理想：你可手动做的几件事

### 1. 调置信度（无需改代码）

在实时监测页把「置信度」在 0.3～0.6 之间试几种，观察误检/漏检变化，选一个折中值。

### 2. 换用更好的自训练权重（推荐）

若你有或能训练出效果更好的权重，可直接替换当前使用的 `best.pt`，无需改代码：

- **抻面**：用新权重替换  
  `models/stretch_detection/weights/best.pt`
- **下面及捞面**：用新权重替换  
  `models/boiling_scooping_detection/weights/best.pt`

替换后重启本服务即可。训练建议：

- 增加**负样本**：背景、窗帘、书架、人脸等「不是手/不是面条」的图，减少误判。
- 手部数据尽量多样：不同角度、光照、遮挡，减少漏检和「脸判成手」。
- 若显存允许，可用更大 backbone（如 yolov8s）再训练一版，替换同上路径。

### 3. 使用官方/外源预训练模型（需类别一致或接受差异）

- **YOLOv8/v11 官方 COCO 预训练**（如 `yolov8n.pt`）  
  - 类别是 person、car 等，**没有** `hand`、`noodle_rope` 等本项目类别，不能直接替代当前「抻面/下面及捞面」的检测逻辑。  
  - 若你只想做「人/手」的通用检测，可以单独做一个入口，用 COCO 模型只检测 person，再配合手部区域裁剪等后处理；这需要改接口与前端，可后续再做。

- **手部专用模型（MediaPipe Hands）**  
  - **已集成**：在实时监测页勾选「手部 MediaPipe：开启（推荐）」即可。手部由 MediaPipe 检测，其他类别仍用 YOLO。  
  - 安装：`pip install mediapipe`。

- **其他公开 hand / 食品检测权重**  
  - 若找到与当前类别名一致的 YOLO 权重（如含 `hand`、`noodle_rope` 等），可放到上述 `weights` 目录并改名为 `best.pt` 替换，重启服务即可。  
  - 若类别名不一致，需要改 `model.names` 或训练/转换脚本，属于模型替换的进阶步骤。

### 4. 重新训练当前模型（提升最大，需你参与标注与训练）

若希望从根本提升准确率，建议在现有数据基础上：

1. **补充标注**  
   - 把误检较多的背景、人脸等裁成图，标为「负样本」或单独一类（训练时作负类/忽略）。  
   - 对手部漏检较多的场景多拍多标。

2. **数据增强**  
   - 亮度、对比度、裁剪、缩放等，提高对光照和角度的鲁棒性。

3. **训练配置**  
   - 适当增加 epoch；必要时用 yolov8s 等稍大模型。  
   - 训练完成后，将新 `best.pt` 按「2. 换用更好的自训练权重」替换即可。

训练脚本与数据格式沿用项目内现有 YOLO 训练流程即可。

## 小结

| 你的目标                     | 建议操作 |
|------------------------------|----------|
| 少一点误检（背景/脸当 hand） | 页面上调高「置信度」；或替换/重训模型并加负样本。 |
| 少一点漏检（手没框）         | 适当降低「置信度」；或增加手部数据并重训。       |
| 减少一只手多个框/重叠框      | 已通过 NMS 与同类别去重叠处理，一般无需再操作。  |
| 想用「更官方/外源」模型      | 手部已支持 MediaPipe（页面勾选）；同类别 YOLO 权重可直接替换 `best.pt`。 |
| 大幅提升准确率               | 按上面「重新训练」做数据与训练，再替换 `best.pt`。 |

**推荐组合**：手部用 MediaPipe（页面默认开启）+ 面条等用现有 YOLO；框颜色已按类别固定，观感更稳定。若需进一步优化面条/背景误检，可替换对应 `best.pt` 或按文档重新训练。
