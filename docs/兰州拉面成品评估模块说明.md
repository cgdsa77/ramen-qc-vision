# 兰州拉面成品分析评估模块说明

## 一、模块目标

基于 **拉面成品标准数据集**（`data/raw/拉面成品/` 下的 `lmcp.*.jpg`）建立成品分析评估体系，用于：
- 统一标注与评估维度，形成可复用的标准数据集；
- 后续可训练分类/评估模型，或作为人工抽检的规范。

---

## 二、评估维度定义

### 1. 主评维度：面条质感（必填）

**定义**：成品中面条的视觉与口感相关品质，包括筋道度、软硬、完整性等。

| 取值 | 说明 |
|------|------|
| `excellent`（优秀） | 筋道、软硬适中、无断生、色泽好 |
| `good`（良好） | 整体良好，略有偏差 |
| `fair`（一般） | 可接受，有明显不足 |
| `poor`（较差） | 偏软/偏硬/断生或明显不达标 |
| `bad`（差） | 严重不达标或难以判断 |

**标注建议**：以“筋道、软硬、完整性”为主，结合画面中面条的形态、色泽判断。

---

### 2. 汤型（必填）

| 取值 | 说明 |
|------|------|
| `red`（红汤） | 汤中有辣椒/红油，呈红色或红褐色 |
| `clear`（清汤） | 无辣椒/红油，汤色清或浅色 |

---

### 3. 面条裹辣椒情况（必填）

| 取值 | 说明 |
|------|------|
| `all`（所有） | 面条表面整体裹有辣椒/红油 |
| `partial`（部分） | 面条表面部分裹有辣椒/红油 |
| `none`（无） | 面条表面无辣椒/红油 |

---

### 4. 辅料（可选，按可见性标注）

**牛肉**（`beef`）

| 取值 | 说明 |
|------|------|
| `none` | 未见 |
| `little` | 少量 |
| `normal` | 适量 |
| `more` | 较多 |

**鸡蛋**（`egg`）

| 取值 | 说明 |
|------|------|
| `none` | 未见 |
| `little` | 少量（如半颗蛋） |
| `normal` | 适量（整颗或接近整颗） |
| `more` | 多（多于一颗等） |

**蒜苗碎**（`scallion`）

| 取值 | 说明 |
|------|------|
| `none` | 未见 |
| `little` | 少量 |
| `normal` | 适量 |
| `more` | 较多 |

若画面中无法判断某项辅料，可标为 `unknown`（仅辅料字段允许）。

## 三、标注数据存放与格式

### 目录结构

```
data/
├── raw/
│   └── 拉面成品/
│       ├── lmcp.1.jpg
│       ├── lmcp.2.jpg
│       └── ...
└── scores/
    └── 拉面成品/
        ├── annotations.json    # 汇总表：每张图一条记录（推荐）
        └── (可选) lmcp.1.json  # 单图标注，与汇总二选一或并存
```

### 单条标注记录示例（汇总表内一条）

```json
{
  "image": "lmcp.1.jpg",
  "noodle_quality": "good",
  "soup_type": "red",
  "noodle_chili": "all",
  "beef": "normal",
  "egg": "none",
  "scallion": "normal",
  "notes": ""
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `image` | string | 是 | 文件名，如 `lmcp.1.jpg` |
| `noodle_quality` | string | 是 | 面条质感：excellent / good / fair / poor / bad |
| `soup_type` | string | 是 | 汤型：red / clear |
| `noodle_chili` | string | 是 | 面条裹辣椒情况：all / partial / none |
| `beef` | string | 否 | 牛肉：none / little / normal / more / unknown |
| `egg` | string | 否 | 鸡蛋：none / little / normal / more / unknown |
| `scallion` | string | 否 | 蒜苗碎：none / little / normal / more / unknown |
| `notes` | string | 否 | 备注 |

## 四、推荐工作流程

1. **确定标准子集（可选）**  
   从 `lmcp.1.jpg ~ lmcp.178.jpg` 中选一批（如 50～100 张）覆盖不同汤型、辅料、质感的图，作为“标准数据集”优先标注。

2. **人工标注**  
   - 使用本仓库提供的 **成品标注工具**（见 [成品标注工具使用说明（拉面成品）](./成品标注工具使用说明_拉面成品.md)）：启动 `python start_web.py` 后访问 **http://127.0.0.1:8000/product-annotation** 在网页上逐张标注；  
   - 或按上述 JSON 格式在 `data/scores/拉面成品/annotations.json` 中直接编辑/追加。

3. **校验与一致性**  
   - 可让多人标同一子集，对比一致率；  
   - 定期检查 `annotations.json` 与 `data/raw/拉面成品/` 中图片是否一一对应（有无漏标、多标）。

4. **后续扩展**  
   - **分析统计**：按汤型、辅料、质感分布做统计与可视化。  
   - **模型**：用标注数据训练分类模型（如 ResNet/ViT 多标签分类），预测面条质感、汤型、辅料等。  
   - **抽检规范**：将本说明与标注结果作为线下抽检的评估标准。

---

## 五、与现有项目的关系

- **抻面 / 下面及捞面**：侧重**过程**（动作、关键帧检测与评分）。  
- **拉面成品评估**：侧重**结果**（成品图的属性与质感评估）。  

两者共用同一套 Web 服务与 `data/` 目录规范，成品标注结果存放在 `data/scores/拉面成品/`，便于统一管理与后续扩展。
