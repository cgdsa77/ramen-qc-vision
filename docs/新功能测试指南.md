## 🧪 测试步骤

### 步骤1：访问抻面检测界面

打开浏览器，访问：
```
http://localhost:8000/web/video_detection.html
```

### 步骤2：上传视频并生成评分

1. **选择视频文件**
   - 建议使用 `cm1`、`cm2`、`cm3` 等已标注的视频
   - 或使用其他抻面相关视频

2. **点击"生成评分"按钮**
   - 系统会自动进行检测和评分
   - 评分过程会使用图像特征提取（如果视频路径有效）

3. **查看评分结果**
   - 总体评分（0-5分）
   - 各类别平均分：
     - `noodle_rope`（面条）：厚度、弹性、光泽、完整性
     - `hand`（手部）：位置、动作、角度、协调性
     - `noodle_bundle`（面束）：紧实度、均匀度
   - 评分规则：应显示"基于标准数据集的评分规则和阈值"

### 步骤3：对比效果

**新功能（使用图像特征）的优势：**
- 评分更准确，基于真实的图像特征而非仅置信度
- 能够区分不同质量的面条（粗细、光泽等）
- 能够评估手部位置和角度的合理性

**如果看到以下情况，说明功能正常：**
- 评分结果在 0-5 分范围内
- 各类别都有详细的属性评分
- 不同视频的评分有明显差异（说明特征提取有效）

## 🔍 验证要点

### 1. 特征提取是否生效

查看浏览器控制台（F12），应该看到：
- 没有特征提取相关的错误信息
- 如果有警告"特征提取失败，回退到置信度评分"，说明视频路径可能有问题

### 2. 评分结果合理性

- **面条评分**：
  - 如果面条较粗、有光泽、完整，评分应该较高（4-5分）
  - 如果面条较细、无光泽、有断裂，评分应该较低（1-3分）

- **手部评分**：
  - 如果手部位置合理、角度正确，评分应该较高
  - 如果手部位置偏、角度不对，评分应该较低

- **面束评分**：
  - 如果面束紧实、均匀，评分应该较高
  - 如果面束松散、不均匀，评分应该较低

### 3. 性能表现

- 评分时间：使用图像特征会增加一些处理时间（通常几秒到几十秒，取决于视频长度）
- 如果处理时间过长，可以考虑：
  - 使用较短的视频片段测试
  - 降低视频分辨率

## 🐛 常见问题

### Q1: 评分结果仍然是占位规则？

**A:** 可能原因：
1. Web服务未重启，仍在使用旧代码
   - 解决：重启Web服务
2. 视频路径无效，特征提取失败
   - 解决：检查上传的视频文件是否有效
3. 特征提取模块导入失败
   - 解决：检查 `src/features/image_feature_extractor.py` 是否存在

### Q2: 评分结果不合理？

**A:** 可能原因：
1. 特征提取算法需要调优
   - 解决：根据实际效果调整特征提取参数
2. 评分阈值需要调整
   - 解决：运行 `scripts/build_scoring_rules.py` 重新生成规则

### Q3: 处理速度太慢？

**A:** 解决方案：
1. 使用较短的视频片段
2. 降低视频分辨率
3. 考虑使用GPU加速（如果可用）
