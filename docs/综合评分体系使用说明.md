# 抻面综合评分体系使用说明

## 一、概述

本评分体系结合**关键帧标注评分**和**骨架线数据**，构建权威、可靠的综合评分系统。

### 1.1 评分原理

- **数据融合**：结合人工标注（36个关键帧）和自动检测（骨架线）的优势
- **权重分配**：根据数据源的可靠性动态分配权重
- **插值策略**：非关键帧通过关键帧插值和骨架线评分进行推断

### 1.2 评分公式

详细公式请参考：`docs/抻面综合评分体系.md`

---

## 二、快速开始

### 2.1 提取标准骨架线统计信息（首次使用）

从标准视频（cm1, cm2, cm3）提取骨架线统计信息，建立评分基准：

```bash
python scripts/extract_standard_skeleton_stats.py
```

**输出**：`data/scores/抻面/standard_skeleton_stats.json`

### 2.2 运行综合评分

对单个视频进行评分：

```bash
python scripts/run_comprehensive_scoring.py cm1
```

对所有视频进行评分：

```bash
python scripts/run_comprehensive_scoring.py
```

**输出**：
- 单个视频结果：`reports/comprehensive_scores/comprehensive_score_{video_name}.json`
- 汇总结果：`reports/comprehensive_scores/summary.json`

## 三、评分结果说明

### 3.1 评分维度

#### 手部评分（Hand Score）
- **position（位置）**：0-5分，评估手部位置是否合理
- **angle（角度）**：0-5分，评估手掌角度是否符合标准
- **action（动作）**：0-5分，评估动作规范性
- **coordination（协调性）**：0-5分，评估双手协调性

#### 面条评分（Noodle Rope Score）
- **thickness（粗细）**：0-5分，评估面条粗细均匀度
- **elasticity（弹性）**：0-5分，评估拉伸弹性
- **gloss（光泽）**：0-5分，评估表面光泽
- **integrity（完整程度）**：0-5分，评估面条完整性

### 3.2 评分等级

- **5分（优秀）**：完全符合标准
- **4分（良好）**：基本符合标准，略有偏差
- **3分（一般）**：存在明显偏差，但可接受
- **2分（较差）**：偏差较大，需要改进
- **1分（差）**：严重偏差，不符合标准

### 3.3 结果格式

```json
{
  "video": "cm1",
  "total_frames": 2798,
  "scored_frames": 1145,
  "average_total_score": 4.2,
  "average_hand_score": 4.1,

          },
          ...
        }
      },
      "hand_score": 4.1,
      "total_score": 4.2
    }
  ]
}
```

## 四、权重配置

### 4.1 类别权重

| 类别 | 权重 | 说明 |
|------|------|------|
| hand | 0.4 | 手部动作重要性 |
| noodle_rope | 0.5 | 面条质量最重要 |
| noodle_bundle | 0.1 | 面条束次要 |

### 4.2 数据源权重

不同评分维度，骨架线和标注的权重不同：

| 评分维度 | 骨架线权重 | 标注权重 | 说明 |
|----------|-----------|---------|------|
| position | 0.4 | 0.6 | 标注更可靠 |
| angle | 0.5 | 0.5 | 两者互补 |
| action | 0.6 | 0.4 | 骨架线特征更丰富 |
| coordination | 0.5 | 0.5 | 两者互补 |
| thickness | 0.3 | 0.7 | 标注直接观察 |
| elasticity | 0.4 | 0.6 | 标注更可靠 |
| gloss | 0.0 | 1.0 | 完全依赖标注 |
| integrity | 0.2 | 0.8 | 标注直接观察 |

**权重调整**：可在 `src/scoring/comprehensive_scorer.py` 中修改 `weights` 配置。

## 五、关键帧插值策略

### 5.1 关键帧识别

系统自动识别已标注的关键帧：
- cm1: 9个关键帧
- cm2: 8个关键帧
- cm3: 19个关键帧

关键帧列表存储在：`data/scores/抻面/{video}/key_frames.json`

### 5.2 插值方法

1. **有前后关键帧**：线性插值
2. **只有前关键帧**：使用前关键帧评分（降低置信度）
3. **只有后关键帧**：使用后关键帧评分（降低置信度）
4. **无关键帧**：完全依赖骨架线评分

## 六、验证和优化

### 6.1 验证方法

1. **标准视频验证**：在cm1, cm2, cm3上验证评分准确性
2. **一致性检查**：确保相邻帧评分的一致性
3. **异常检测**：识别并修正异常值

### 6.2 优化方向

1. **权重调整**：根据实际效果调整权重配置
2. **阈值优化**：优化评分阈值，提高准确性
3. **特征增强**：添加更多骨架线特征
4. **插值改进**：改进非关键帧的插值策略

## 七、常见问题

### Q1: 为什么有些帧没有评分？

**A**: 可能原因：
1. 该帧没有检测到手部（骨架线数据为空）
2. 该帧不是关键帧，且无法通过插值推断

**解决方案**：
- 检查骨架线数据是否完整
- 增加关键帧标注数量

### Q2: 如何提高评分准确性？

**A**: 建议：
1. 增加关键帧标注数量（特别是动作变化明显的帧）
2. 优化骨架线检测（提高召回率）
3. 调整权重配置（根据实际效果）

### Q3: 评分结果与预期不符？

**A**: 可能原因：
1. 权重配置不合理
2. 标准数据集统计信息不准确
3. 骨架线特征提取有误

**解决方案**：
- 检查标准骨架线统计信息
- 调整权重配置
- 验证骨架线特征提取逻辑

## 八、技术细节

### 8.1 骨架线特征提取

主要特征：
- 手腕位置（wrist_x, wrist_y）
- 手掌角度（palm_angle）
- 手指张开度（finger_spread）
- 双手距离（hand_distance）
- 双手对称性（symmetry_y_diff）
- 动作速度（velocity_magnitude）
- 拉伸速度（stretch_velocity）

### 8.2 评分计算流程

1. **提取骨架线特征**：从MediaPipe关键点提取特征
2. **计算骨架线评分**：基于特征和标准统计信息计算评分
3. **加载标注评分**：从关键帧标注文件加载评分
4. **综合评分**：根据权重融合骨架线评分和标注评分
5. **插值处理**：对非关键帧进行插值
6. **异常检测**：检测并修正异常值
