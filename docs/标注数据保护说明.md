# 标注数据保护说明

## 问题描述

用户反馈在使用labelImg等标注工具时，有时会发现标注文件的类别被意外修改（如noodle_rope被改成noodle_bundle）。

## 可能的原因

1. **标注工具的classes.txt顺序不一致**: 不同视频目录的classes.txt顺序可能不同，导致类别ID映射错误
2. **标注工具重新保存**: labelImg等工具在保存时可能会重新映射类别ID
3. **脚本误操作**: 运行了可能修改标注的脚本（如fix_all_labels.py）

## 保护措施

### 1. 备份标注数据

在修改标注或运行训练前，建议先备份：

```bash
python scripts/backup_labels.py
```

这会创建一个带时间戳的备份，保存在 `data/labels_backup/` 目录下。

### 2. 检查标注一致性

检查标注文件是否被修改：

```bash
python scripts/check_label_consistency.py
```

首次运行会创建校验和文件，后续运行会检查文件是否被修改。

更新校验和（在确认标注正确后）：

```bash
python scripts/check_label_consistency.py save
```

### 3. 检查标注统计

查看每个视频的类别分布，确认标注是否正确：

```bash
python scripts/check_label_issues.py
```

### 4. 使用标注工具时的注意事项

1. **统一classes.txt顺序**: 
   - 确保 `data/labels/抻面/classes.txt` 的顺序正确
   - 确保各视频子目录的classes.txt顺序一致

2. **使用labelImg时**:
   - 打开labelImg后，先检查 "Open Dir" 和 "Change Save Dir" 是否正确
   - 检查 "View" -> "Auto Save" 是否启用
   - 保存前确认类别名称正确

3. **导出/导入标注时**:
   - 使用MakeSense.ai等在线工具时，确保导出时的类别顺序正确
   - 导入前检查类别映射是否正确

## 如果发现标注被修改

1. **检查备份**: 查看 `data/labels_backup/` 目录，找到最近的备份
2. **恢复备份**: 从备份中恢复被修改的文件
3. **重新检查**: 运行 `check_label_consistency.py` 确认恢复正确
4. **更新校验和**: 运行 `check_label_consistency.py save` 更新校验和

## 预防措施

1. **定期备份**: 在大量修改标注前，先运行备份脚本
2. **使用校验和**: 定期运行一致性检查，及早发现问题
3. **版本控制**: 可以考虑使用Git等版本控制系统管理标注文件（如果文件不是太大）
