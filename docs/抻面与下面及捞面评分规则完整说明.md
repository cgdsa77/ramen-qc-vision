# 抻面与下面及捞面 评分规则完整说明

本文档仅包含**抻面**、**下面及捞面**两套评分规则，涵盖规则文件结构、各属性含义、阈值、权重、计算流程及后续可优化点，便于后续修改与优化。不涉及拉面成品。

---

## 一、规则文件位置与结构

| 环节       | 规则文件路径 |
|------------|--------------|
| 抻面       | `data/scores/抻面/scoring_rules.json` |
| 下面及捞面 | `data/scores/下面及捞面/scoring_rules.json` |

每个 JSON 均包含以下顶层键：

- **version**：规则版本（如 "1.0"）。
- **stage**：环节标识（"抻面" / "下面及捞面"）。
- **thresholds**：各属性的等级阈值（excellent / good / fair / poor，及 mean、std），用于将连续值映射为 1～5 分。
- **statistics**：历史标注统计（mean、std、min、max、distribution 等），供分析与校准用。
- **weights**：每个**检测类别**下各**属性**的子权重（该类内权重和为 1）。
- **overall_weights**：各**检测类别**在**总分**中的权重（即「评分标准」弹窗中可调项，总和须为 1）。

## 二、抻面（Stretch）评分规则

### 2.1 检测类别与 overall_weights（维度权重）

培训师在「评分标准」中调整的即下表三项，总和须为 100%。

| 键名          | 界面名称 | 默认权重 | 说明 |
|---------------|----------|----------|------|
| noodle_rope   | 面绳     | 0.6 (60%) | 面条本体：粗细、弹性、光泽、完整性 |
| hand          | 手部     | 0.3 (30%) | 手部动作：位置、动作、角度、协调 |
| noodle_bundle | 面束     | 0.1 (10%) | 成束状态：紧实度、均匀度 |

### 2.2 各类别下属性与 weights（子权重）

- **noodle_rope（面绳）**

| 属性 attr    | 子权重 | 含义 |
|-------------|--------|------|
| thickness   | 0.3    | 厚度/粗细 |
| elasticity  | 0.25   | 弹性 |
| gloss       | 0.2    | 光泽 |
| integrity   | 0.25   | 完整性 |

- **hand（手部）**

| 属性 attr    | 子权重 | 含义 |
|-------------|--------|------|
| position    | 0.25   | 手部位置 |
| action      | 0.3    | 动作规范 |
| angle       | 0.2    | 角度 |
| coordination| 0.25   | 协调性 |

- **noodle_bundle（面束）**

| 属性 attr  | 子权重 | 含义 |
|-----------|--------|------|
| tightness | 0.5    | 紧实度 |
| uniformity| 0.5    | 均匀度 |

### 2.3 抻面 thresholds（等级阈值）

每个属性在 `thresholds` 中有四个等级阈值（excellent / good / fair / poor）及 mean、std。属性值 ≥ excellent 为 5 分，≥ good 为 4 分，≥ fair 为 3 分，≥ poor 为 2 分，否则 1 分。

| 属性         | excellent | good  | fair  | poor  | mean(约) | std(约) |
|--------------|-----------|-------|-------|-------|----------|---------|
| gloss        | 4.5       | 3.5   | 2.5   | 2.06  | 3.5      | 0.72    |
| integrity    | 4.5       | 3.5   | 2.5   | 2.11  | 3.62     | 0.75    |
| elasticity   | 4.5       | 3.5   | 2.5   | 2.01  | 3.75     | 0.87    |
| thickness    | 4.5       | 3.5   | 2.73  | 2.38  | 3.77     | 0.69    |
| coordination | 4.5       | 3.5   | 2.86  | 2.54  | 3.81     | 0.63    |
| angle        | 4.5       | 3.64  | 3.0   | 2.68  | 3.95     | 0.64    |
| action       | 4.5       | 3.73  | 2.97  | 2.59  | 4.11     | 0.76    |
| position     | 4.5       | 3.66  | 3.15  | 2.90  | 3.92     | 0.51    |
| tightness    | 4.5       | 3.81  | 3.49  | 3.33  | 3.97     | 0.32    |
| uniformity   | 4.5       | 3.5   | 2.77  | 2.40  | 3.86     | 0.73    |

（具体数值以当前 `data/scores/抻面/scoring_rules.json` 为准。）

### 2.4 抻面评分计算流程

1. **单检测框**：按类别（noodle_rope / hand / noodle_bundle）提取图像特征或使用置信度，得到各属性原始值 → 经 `thresholds` 映射为 1～5 分 → 按 `weights[class_name]` 加权得到该类**加权分**。
2. **单帧**：该帧所有检测框按类别聚合并求类内平均加权分，得到 `class_scores`；再按 `overall_weights` 加权得到该帧 **overall_score**。
3. **视频**：仅对「有检测到的类别」参与总分；`overall_weights` 只对当前帧出现的类别归一化后加权；视频级总分 = 各帧 overall 的平均（或按「有出现的类别」的 class_avg 再按 overall_weights 归一化，与实现一致）。
4. **代码中的偏置**：抻面 `StretchScorer` 在 `calculate_weighted_score` 中对 hand 加 0.7、noodle_rope 加 0.8 的线性偏置，再裁到 [1, 5]，使自动分更接近人工均值。

### 3.4 下面及捞面评分计算流程

1. **单检测框**：对 noodle_rope / hand / tools_noodle / soup_noodle 四类，有图像则用图像特征 + 校准 + `thresholds` 映射为 1～5 分；无图像或失败则用置信度生成各属性分。再按 `weights[class_name]` 加权得到该类**加权分**。
2. **单帧**：该帧所有检测按类别聚合并求类内平均加权分 → 按 `overall_weights` 仅对**本帧出现的类别**归一化后加权，得到该帧 **overall_score**。
3. **视频**：仅「有检测到的类别」参与；视频级总分 = 按 `overall_weights` 对各类别平均分加权并重新归一化（仅对出现类别）；`noodle_bundle` 若需展示，由 tools_noodle 与 soup_noodle 平均得到（操作规范）。
4. **特征来源**：tools_noodle 使用手部相关特征代理（operation_standardization、tool_coordination）；soup_noodle 使用面条特征代理（ripeness、distribution_state）；noodle_rope 若缺 ripeness/soup_adhesion 等则由 elasticity/integrity/gloss/thickness 等映射补全。

## 五、总分公式小结

- **单帧总分（两套规则通用逻辑）**  
  设该帧出现的类别集合为 \( C \)，各类别在该帧的平均加权分为 \( S_c \)，则：
  \[
  \text{overall\_score} = \frac{\sum_{c \in C} S_c \cdot W_c}{\sum_{c \in C} W_c},
  \]
  其中 \( W_c \) 为 `overall_weights[c]`。

- **视频总分**  
  - 抻面：对「有出现的类别」的类别平均分再按 `overall_weights` 归一化后加权。  
  - 下面及捞面：同上；`noodle_bundle` 仅用于展示，由 tools_noodle 与 soup_noodle 平均得到。

## 六、后续可优化与修改方向

1. **overall_weights**  
  在「评分标准」弹窗中已支持培训师修改；修改后写入对应 `scoring_rules.json`，下次评分即生效。

2. **weights（类别内子权重）**  
  当前在 JSON 中写死；若需「面绳更看重弹性、手部更看重动作」等，可改 `data/scores/抻面/scoring_rules.json` 与 `data/scores/下面及捞面/scoring_rules.json` 中的 `weights`，或后续为培训师提供界面编辑。

3. **thresholds**  
   决定「多少分算优秀/良好/一般/差」；可根据新标注数据重新统计 mean/std 与分位数，用 `scripts/build_scoring_rules.py`、`scripts/build_boiling_scooping_scoring_rules.py` 等重新生成或手调 thresholds。

4. **抻面 hand/noodle_rope 偏置**  
   已改为**动态偏置**：仅在 1.2＜分数＜4.8 时施加，且越接近两端偏置越小；基准仍为 hand +0.7、noodle_rope +0.8。若需关闭或调整，可改 `StretchScorer.calculate_weighted_score`。

5. **下面及捞面「仅用有检测到的类别」**  
   已实现；若希望未出现类别按 0 分参与而非排除，可改 `score_video` 中 present_classes 与归一化逻辑。

6. **新增属性或类别**  
   需同时修改：规则 JSON（thresholds、weights、overall_weights）、评分器中对 `weights`/类别的引用、以及（若用图像特征）特征提取与校准逻辑。

## 七、已实施的优化（对应《抻面与下面及捞面评分规则全体系优化方案》）

以下内容已按方案落地，与原有框架兼容：

- **阈值**：下面及捞面规则中无区分度阈值已修正为严格单调（excellent＞good＞fair＞poor），数值保留 2 位小数；校验脚本见 `scripts/validate_scoring_rules.py`，修正脚本见 `scripts/fix_boiling_thresholds.py`。
- **偏置**：抻面固定偏置改为分段动态偏置，仅对中间分数区间修正，避免极端值失真。
- **边界**：仅置信度≥0.3 的检测框参与评分；视频级「有效参与类别」为该类别在至少 **10%** 帧中出现（兼顾 xl1 等面条出现较少的视频仍能出分）；有效评分帧数＜总帧数 50% 时返回 `warning` 提示。
- **代理特征**：下面及捞面 `tools_noodle`/`soup_noodle` 在置信度＜0.5 时加权分×0.9，＜0.3 时×0.8。
- **可追溯**：视频评分结果中增加 `normalization_detail`（有效类别、归一化权重、各类别平均分）与 `warning`，便于核对与排查。

未在代码中实现、需后续或线下完成的部分：基于标注数据的分位数/回归校准、多套阈值/权重模板与界面切换、关键帧加权、灰度发布与回滚等，见《全体系优化方案》第二～五阶段。
