# 紧急问题：标注被错误替换

## 问题描述

用户在labelImg中只对一个图片新增了noodle_bundle标注，但是发现所有原来的noodle_rope都被替换成了noodle_bundle。

## 根本原因

这是labelImg的一个已知问题：当labelImg保存标注时，如果classes.txt的顺序或内容发生变化，或者labelImg内部的状态不一致，可能会导致类别ID映射错误。

## 可能的原因

1. **classes.txt顺序问题**: labelImg读取classes.txt时，如果顺序与标注时的顺序不同，会导致类别ID映射错误
2. **labelImg缓存问题**: labelImg可能在内存中缓存了错误的类别映射
3. **保存时的类别映射错误**: labelImg在保存时可能使用了错误的类别ID映射

## 解决方案

### 1. 立即修复（已完成）

运行修复脚本：
```bash
python scripts/emergency_fix_cm1_labels.py
```

这个脚本会：
- 自动备份cm1的所有标注文件
- 将错误的noodle_bundle (ID=2) 改回 noodle_rope (ID=1)
- 显示修复的文件和标注数量

### 2. 验证修复结果

```bash
python scripts/emergency_check_cm1.py
```

### 3. 预防措施

#### 使用labelImg时的注意事项：

1. **每次打开前检查classes.txt**:
   - 确保 `data/labels/抻面/classes.txt` 的顺序正确
   - 顺序必须是：hand, noodle_rope, noodle_bundle

2. **使用labelImg前的检查清单**:
   - [ ] 运行 `python scripts/backup_labels.py` 备份
   - [ ] 检查classes.txt顺序是否正确
   - [ ] 检查"Open Dir"和"Change Save Dir"是否正确

3. **使用labelImg时的操作**:
   - 只打开一个视频目录（如cm1）
   - 不要同时打开多个目录
   - 保存后立即检查几个文件是否正确

4. **保存后验证**:
   - 运行 `python scripts/check_label_consistency.py` 检查是否有修改
   - 随机检查几个标注文件，确认类别ID正确

## 长期解决方案

### 1. 使用更稳定的标注工具

考虑使用X-AnyLabeling或MakeSense.ai，这些工具对类别映射的处理更可靠。

### 2. 定期备份和检查

- 每次修改标注前都备份
- 使用校验和定期检查标注文件

### 3. 使用脚本验证

创建一个简单的验证脚本，在使用labelImg前后运行，确保标注没有被错误修改。

## 如果问题再次发生

1. **立即停止使用labelImg**
2. **检查备份**: `data/labels_backup/` 目录
3. **运行修复脚本**: 根据具体情况修改修复脚本
4. **报告问题**: 记录哪些文件被修改，以便找出规律

## 建议

鉴于labelImg的稳定性问题，强烈建议：
1. 在修改标注前始终备份
2. 考虑切换到更稳定的标注工具（如X-AnyLabeling）
3. 使用版本控制系统管理标注文件（如果文件数量不是太大）
