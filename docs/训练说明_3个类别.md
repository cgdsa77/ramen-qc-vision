# 使用3个类别重新训练模型

## 训练配置

### 类别设置
- **hand** (类别ID: 0)
- **noodle_rope** (类别ID: 1)
- **noodle_bundle** (类别ID: 2)

已从`classes.txt`中移除未使用的`pot_or_table`类别。

### 训练数据
- **视频列表**: cm1, cm2, cm3, cm4, cm5, cm6, cm7
- **总标注框数**: 2445个
  - hand: 857个 (35.1%)
  - noodle_rope: 1119个 (45.8%)
  - noodle_bundle: 469个 (19.2%)

### 训练参数
- **模型**: YOLOv8n (轻量级，训练速度快)
- **训练轮数**: 150 epochs
- **批次大小**: 8
- **图片尺寸**: 640x640
- **早停耐心值**: 50轮
- **验证集划分**: 20% (使用最后几个视频)
- **设备**: CPU (避免CUDA兼容性问题)

### 数据增强
- HSV色调/饱和度/明度增强
- 旋转、平移、缩放
- 左右翻转
- Mosaic、MixUp增强

## 训练命令

```bash
python src/training/train_detection_model.py --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 --epochs 150 --batch 8 --no-resume
```

使用`--no-resume`从头开始训练，避免使用之前错误的4类别配置。

## 查看训练进度

运行以下命令查看训练进度：
```bash
python scripts/check_training_progress.py
```

或双击：`查看训练进度.bat`

## 训练完成后

1. 最佳模型会保存在：`models/stretch_detection/weights/best.pt`
2. 模型会自动复制到：`models/stretch_detection_model.pt`
3. 重启Web服务器以使用新模型
4. 测试检测效果，观察识别准确率是否提升

## 预期改进

使用正确的3类别配置训练后，预期会：
- ✅ 消除类别混淆（不再有未使用的pot_or_table类别）
- ✅ 提高类别识别准确率
- ✅ 减少hand和noodle_rope的识别错误
- ✅ 模型更专注于3个目标类别的学习

## 注意事项

如果训练后效果仍不理想：
1. 检查标注质量，特别是hand和noodle_rope的边界
2. 考虑继续标注剩余视频（cm8-cm12）
3. 可以调整训练参数（学习率、数据增强等）
