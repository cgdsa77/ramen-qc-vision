## 问题1: 是否继续标注剩余的抻面视频？

### 建议：**可以，但非必需**

#### 继续标注的优缺点：

**优点：**
1. ✅ **数据多样性提升**：更多视频可以增加数据集的多样性，提升模型泛化能力
2. ✅ **提升mAP50**：更多训练数据通常能进一步提升模型性能（可能达到0.75+）
3. ✅ **覆盖更多场景**：cm8~cm12可能包含不同的场景、角度、光照条件

**缺点/考虑：**
1. ⚠️ **时间成本**：标注5个视频需要大量时间
2. ⚠️ **边际收益**：当前mAP50=0.6940已经很好，进一步提升可能需要更多数据才能看到明显效果
3. ⚠️ **效果验证**：如果当前模型在实际使用中效果已经满足需求，可能不需要继续标注

### 推荐策略：

**方案A（推荐）**: **先验证当前模型效果，再决定**
- 使用当前模型（mAP50=0.6940）在实际场景中测试
- 如果效果已经很好，可以跳过剩余标注，直接开始下面/捞面部分
- 如果发现某些场景识别不佳，再针对性地标注cm8~cm12

**方案B**: **快速标注部分视频**
- 如果时间允许，可以标注2-3个视频（如cm8, cm9, cm10）
- 快速增加数据量，重新训练一次看效果
- 如果提升不明显，就停止标注

**方案C**: **完整标注所有视频**
- 如果目标是追求最高精度，可以标注所有剩余视频
- 预期mAP50可能提升到0.75~0.80

### 下面（煮面）部分的类别建议

建议检测的类别：
- `pot` - 锅（识别是否在锅里）
- `noodle_in_pot` - 锅中的面条
- `hand` - 手部（如果需要）
- `water_boiling` - 沸腾状态（可选，较难检测）
- `foam` - 泡沫（可选，识别是否溢锅）

**推荐类别（3-4类）**：
```

noodle_in_pot
hand
```

- `strainer` - 漏勺/捞面工具
- `pot` - 锅

**推荐类别（3-4类）**：

## 训练模型的基础

当需要继续训练抻面模型时，可以使用以下命令：

```bash
# 使用当前最佳模型作为预训练基础
python src/training/train_detection_model.py \
    --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 cm8 cm9 cm10 \
    --epochs 200 \
    --batch 8 \
    --patience 30 \
    --no-resume  # 如果想从头开始训练，或使用 --resume 从检查点继续
```

**重要**：当前最佳模型已保存，任何新的训练都不会覆盖它，除非你明确复制新的best.pt文件。
