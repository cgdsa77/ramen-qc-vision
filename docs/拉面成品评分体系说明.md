# 拉面成品评分体系说明

## 一、设计原则

- **以面条质感为主体**：总分由面条质感（筋道、软硬、完整性、色泽）主导，不单一依赖食材有无。
- **汤型与辣椒不参与惩罚**：清汤与红汤、面条裹辣椒（all/partial/none）仅反映汤型或进食状态，**不**作为扣分依据。例如：
  - 清汤成品与红汤成品在同等质感下得分一致；
  - 面条无辣椒（清汤或未拌开）不扣分。
- **辅料仅作可选小幅加分**：牛肉/鸡蛋/蒜苗碎仅作 0～5 的呈现加分，无辅料时得 0，**不扣分**。

## 二、评分公式

```
S_final = S_texture + w_presentation × S_presentation
```

- **S_texture**：由 `noodle_quality` 唯一决定，取值见下表。
- **S_presentation**：0～5，仅对辅料丰富度做小幅正向加分，无辅料为 0。
- **w_presentation**：0.04（约 2 分上限），保证质感主导。
- 结果限制在 **[0, 100]**。

### 面条质感与 S_texture 映射

| noodle_quality | 说明           | S_texture |
|----------------|----------------|-----------|
| excellent      | 优秀           | 92        |
| good           | 良好           | 77        |
| fair           | 一般           | 58        |
| poor           | 较差           | 35        |
| bad            | 差             | 15        |

### 呈现加分 S_presentation（可选）

- **依赖**：由标注或用户勾选的 **牛肉、鸡蛋、蒜苗碎** 三项计算，无辅料时为 0，不扣分。
- 牛肉：none=0, little=0.5, normal=1, more=1.5  
- 鸡蛋：none=0, yes=1, little=0.5  
- 蒜苗碎：none=0, little=0.5, normal=1, more=1.5  
- 上限 5 分。**上传图片评分**时，模型只预测面条质感，不识别辅料；在成品综合评分页的「辅料（用于呈现加分）」中勾选牛肉/鸡蛋/蒜苗后，会按上述规则重算呈现加分并更新报告。

## 三、规则与模型文件

- **规则文件**：`data/scores/拉面成品/product_scoring_rules.json`
- **标注汇总**：`data/scores/拉面成品/annotations.json`
- **训练得到的模型**（可选）：`data/scores/拉面成品/product_model/product_model.joblib`、`config.json`

## 四、使用方式

### 1. 仅用标注计算得分（规则公式）

对已有标注直接算分，不跑图像模型：

```python
from src.scoring.product_scorer import ProductScorer, load_annotations

scorer = ProductScorer()
# 单条
out = scorer.score_from_annotation({"noodle_quality": "good", "soup_type": "clear", "noodle_chili": "none", ...})
# 批量
items = load_annotations()
results = scorer.batch_score_from_annotations(items)
```

### 2. 训练成品质感预测模型（融合多方法）

基于现有标注与图片，训练「面条质感」分类器（不依赖汤型/辣椒）：

```bash
# 在项目根目录
python -m src.training.train_product_model
```

- 特征：ResNet18 全局池化特征 + 颜色直方图/边缘密度等手工特征  
- 模型：RandomForest + LogisticRegression 软投票融合  
- 输出：`data/scores/拉面成品/product_model/` 下 `product_model.joblib`、`config.json`  

可选参数：`--no-resnet`（仅手工特征）、`--no-hand`（仅 ResNet）、`--out-dir` 指定输出目录。

**手工特征维度（20 维 vs 27 维）**  
- **27 维**：24 维 BGR 颜色直方图 + 3 维边缘密度/亮度/对比度，信息更完整，**推荐**。训练脚本已统一为 27 维，并在中文路径下通过临时英文路径重试读图，保证训练时也能用到真实手工特征。  
- **20 维**：多为早期训练时因读图失败得到的零向量（512+20=532），模型未利用到手工信息。预测器对已保存的 532 维模型做了兼容（自动截取 20 维）。  
- **建议**：重新运行一次 `python -m src.training.train_product_model`，得到 539 维（512+27）模型后，效果会优于仅用 ResNet 或 20 维零向量的旧模型。

### 3. 对单张图片预测并打分

训练完成后，对未标注图片预测质感并得到 0～100 分：

```python
from src.scoring.product_predictor import ProductPredictor

predictor = ProductPredictor()
result = predictor.predict_and_score(Path("data/raw/拉面成品/lmcp.1.jpg"))
# result: noodle_quality, total_score, s_texture, confidence, ...
```

### 4. HTTP API（需启动 start_web.py）

- **POST /api/product-score**  
  - Body: `{"annotation": {...}}` 或 `{"items": [...]}`  
  - 返回规则公式下的得分。

- **GET /api/product-score/batch**  
  - 对当前 `annotations.json` 全部标注批量算分。

- **GET /api/product-score/image/{filename}**  
  - 对 `data/raw/拉面成品/{filename}` 用训练好的模型预测并打分（需先训练）。

## 五、依赖

- 规则评分：仅需项目内 `product_scoring_rules.json`。
- 训练脚本：`torch`、`torchvision`、`scikit-learn`、`Pillow`、`opencv-python`。
- 预测（图像模型）：同上；模型文件需已存在。

## 六、与标注维度的关系

标注中的 **soup_type**、**noodle_chili** 仅用于记录与后续分析，**不参与 S_texture 与 S_final 的计算**，从而避免「清汤不如红汤」「无辣椒就扣分」等死板逻辑。成品得分以**面条质感**为主，符合真实品控场景。
