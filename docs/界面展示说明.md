# 兰州拉面质检系统 — 界面展示说明文档

本文档描述当前 Web 主界面的结构、角色区分、主题与动效，便于其他 AI 或开发者在此基础上进行优化。

---

## 一、项目与系统简介

- **系统名称**：面向兰州拉面制作的标准化流程视觉质检系统  
- **主入口**：单页应用，登录后根据角色展示不同主题与文案的主 dashboard  
- **主要功能**：抻面/下面及捞面的流程检测、标注工具、综合评分、姿态视频展示；管理员另有用户管理与系统设置，培训师另有学员管理与评分标准配置  

---

## 二、技术栈与文件位置

- **主界面文件**：`web/index.html`（HTML + 内联 CSS + 内联 JS，无前端框架）  
- **字体**：Google Fonts — Noto Serif SC、Noto Sans SC  
- **动效库**：Popmotion（已集成），通过 `window.popmotion` 或 `popmotion.bundle.js` 引入，用于登录/主界面动画与卡片悬停等  
- **后端**：FastAPI（`start_web.py`），会话与权限通过 `/api/auth/session`、`/api/auth/login` 等接口维护

## 三、页面结构概览

### 3.1 两种整体状态

| 状态       | body 类名              | 说明 |
|------------|------------------------|------|
| 未登录     | `login-page`           | 显示登录弹窗；背景为图片 + 半透明遮罩；左右两侧为拉面成品滚动播放带（仅登录页显示） |
| 已登录     | `main-page` + `role-0` / `role-1` / `role-2` | 显示主界面 `#mainWrap`，包含顶条、导航、Hero、主内容区、页脚；主题与文案按角色切换 |

### 3.2 登录页（未登录时）

- **背景**：`/data/web_background_images/web1_bg.jpg` + 深色渐变遮罩，`background-size: contain`，固定定位  
- **左右播放带**：`.side-panel.left` / `.side-panel.right`，各 3 列 `.side-strip`，每列内为 `.side-strip-track` + 多个 `.side-strip-item`（竖条比例 `aspect-ratio: 2/3`），图片来自接口 `/api/product-images/list`；悬停时暂停滚动  
- **登录弹窗**：`#loginModal`，居中模态，含账号、密码、登录按钮、错误提示、忘记密码说明（无关闭按钮，必须登录）  

### 3.3 主界面（已登录后）

结构自上而下：

1. **顶条** `.top-bar`  
   - 高度 4px，渐变条；角色主题会覆盖其 `background`  
   - 入场动画：`body.main-page .top-bar` 使用 `topBarReveal`（scaleX 0→1，约 0.6s）

2. **导航栏** `.nav-bar`  
   - 左侧：`#navUserInfo`（当前用户：姓名/账号（角色名））  
   - 右侧按钮：用户管理(role-0)、系统设置(role-0)、学员管理(role-1)、评分标准(role-1)、退出(全体)  
   - 通过 `data-role-visible="0"` 等控制显示，由 `applyRoleVisibility()` 根据 `currentUser.role` 显隐  

3. **Hero** `.hero`  
   - 主标题：`<h1>` 固定为「面向兰州拉面制作的标准化流程视觉质检系统」  
   - 副标题：`#heroSubtitle`，文案由角色通过 `ROLE_COPY[role].heroSubtitle` 注入  
   - 背景为渐变 + 装饰纹理（::before 的 SVG 图案）；角色主题会覆盖 `.hero` 的 `background`  

4. **主内容区** `.container`  
   - 最大宽度约 1100px，居中；背景由角色主题覆盖  
   - 内含多个 **区块** `.section`，每个有标题 `.section-title` 和可选下划线动效 `.section-title-line`  

5. **页脚** `.footer`  
   - 一句系统说明；文字颜色由角色主题覆盖

## 四、角色体系与权限

- **角色码**：`0` = 管理员，`1` = 培训师，`2` = 厨师/学员  
- **登录成功后**：`currentUser.role` 决定 body 的 `role-0` / `role-1` / `role-2`，以及：  
  - 导航按钮的显隐（`data-role-visible`）  
  - 主界面各 section、各 card-grid 的显隐  
  - 副标题、区块标题、卡片标题与描述（`applyRoleCopy(role)` 从 `ROLE_COPY` 写入）  
  - 整页配色与背景（见下节）

### 5.3 厨师/学员（role-2）

## 六、主内容区模块结构

### 6.1 区块（sections）与可见角色

| 区块 id             | 默认标题（可被 ROLE_COPY 覆盖） | 可见角色   |
|---------------------|----------------------------------|------------|
| section-process     | 流程检测                         | 0, 1, 2    |
| section-annotation | 标注工具                         | 0, 1       |
| section-scoring     | 综合评分                         | 0, 1, 2    |
| section-pose        | 姿态与视频展示                   | 0, 1       |

- 每个 section 的标题写在 `<span id="title-{process|annotation|scoring|pose}">` 中，由 `applyRoleCopy` 按角色替换文案。  

### 6.2 卡片（cards）与 data-card

卡片在 `.card-grid` 内，每个卡片有 `data-card="xxx"`，用于从 `ROLE_COPY[role].cards.xxx` 取 `title`、`desc` 写进 `<h3>` 和 `<p>`。

**流程检测（section-process）**

- `stretch` — 抻面动作检测，链向 `/stretch-detection`  
- `boiling` — 下面及捞面检测，链向 `/boiling-detection`  
- `realtime` — 实时监测，链向 `/realtime-monitor`  

**标注工具（section-annotation）**

- `annotate-stretch` — 抻面评分标注工具，链向 `/scoring-tool`  
- `annotate-boiling` — 下面及捞面评分标注工具，链向 `/scoring-tool-boiling`  
- `annotate-product` — 兰州拉面成品评估标注，链向 `/product-annotation`  

**综合评分（section-scoring）**

- 对 role 0、1 显示：`score-stretch`、`score-boiling`、`score-product`（分别链向综合评分可视化、下面及捞面可视化、成品综合评分）  
- 对 role 2 显示：`chef-history`（我的历史记录，链向 `/scoring-visualization`）  

**姿态与视频（section-pose）**

- `pose-stretch` — 抻面手部姿态视频，链向 `/video-skeleton`  
- `pose-boiling` — 下面及捞面手部姿态视频，链向 `/video-skeleton-boiling`

## 七、角色文案配置（ROLE_COPY）

在 JS 中存在对象 `ROLE_COPY`，结构为：

```js
ROLE_COPY[role] = {
  heroSubtitle: '...',           // 副标题一行
  sections: {
    'section-process': '...',
    'section-annotation': '...',
    'section-scoring': '...',
    'section-pose': '...'
  },
  cards: {
    'stretch': { title: '...', desc: '...' },
    'boiling': { ... },
    // ... 其他 data-card 键
  }
}
```

- 管理员(0)：副标题与卡片用语偏「监管、数据管理、报告、资源」  
- 培训师(1)：偏「教学、学员、评分标准、示范」  
- 厨师/学员(2)：偏「我的、个人、练习、历史记录」  

优化时若要改某角色下某模块的「名称/描述」，只需改 `ROLE_COPY` 中对应字段，无需改 HTML（卡片本身仍通过 `data-card` 与 `ROLE_COPY.cards` 绑定）。

## 九、动效说明

### 9.1 CSS

- **顶条入场**：`body.main-page .top-bar` 的 `topBarReveal`（scaleX + opacity）  
- **区块标题下划线**：`.section-title-line` 的宽度在进入视口时由 0 动画到 100%（通过 JS IntersectionObserver 触发 Popmotion）  

### 9.2 Popmotion（runPageAnimations）

- **Hero**：主标题逐字 opacity + translateY 入场；副标题延迟淡入  
- **卡片**：依次 opacity + translateY 入场（带 stagger）；随后绑定 **鼠标悬停**：按角色使用不同上浮量（role-0: -6px，role-1: -10px，role-2: -8px）和弹簧参数，实现卡片上浮与阴影变化  
- **按钮**（卡片内 .btn、.btn-nav、.btn-primary）：悬停/按下时 scale 弹簧动效  

### 9.3 登录页

- `runLoginAnimations`：登录框淡入 + 位移动画；输入框 focus 时轻微上浮与阴影（若使用 Popmotion 则同上）。

## 十一、可优化方向建议（供后续 AI 参考）

1. **视觉与布局**  
   - 各角色主题可再微调配色、对比度、圆角与阴影，使三种身份更易区分且不刺眼。  
   - 卡片网格在窄屏下的响应式（栅格列数、间距、字体大小）。  
   - Hero 在不同分辨率下的字号与副标题折行。  

2. **动效**  
   - 可增加区块进入视口时的淡入/上浮，或卡片 stagger 更明显。  
   - 弹窗打开/关闭用 Popmotion 做 scale + opacity，替代单纯 display。  
   - 顶条、导航栏与角色主题的过渡可考虑更顺滑（如主题切换时的短暂过渡）。  

3. **无障碍与性能**  
   - 为图标按钮、弹窗关闭等补充 `aria-label` 或说明文字。  
   - 减少或延迟非首屏动画，避免过多同时运行的 Popmotion 实例。  

4. **结构与可维护性**  
   - 若后续扩展角色或模块，可将 `ROLE_COPY` 与主题色提取为独立配置文件或 JSON，便于非前端修改文案与主题。  
   - 弹窗较多时，可考虑统一用 data 属性或 JS 配置「打开哪个弹窗、关闭时是否回到上一级」等，减少重复绑定。  

5. **一致性**  
   - 弹窗内按钮（如「保存」「评分标准说明」）与主界面 `.card .btn` 的样式可统一为设计系统内的一组主按钮/次按钮。  
   - 各角色下表单输入框、表格、说明文字的字号与行高可统一成一套规范。
