# 姿态估计当前状态和解决方案

## 当前问题

已安装MediaPipe 0.10.31，但该版本的API与旧版本（<0.10）发生了重大变化：
- **旧版本API**：`mediapipe.solutions.pose` ✅ 代码中使用的方式
- **新版本API**：`mediapipe.tasks.python` ❌ 需要重写代码

## 解决方案

### 方案1：使用MMPose（推荐）

MMPose是更专业的姿态估计框架，功能强大且稳定：

```bash
# 安装MMPose
pip install mmpose mmcv

# 使用RTMPose模型（轻量级、效果好）
```

**优点**：
- 更专业的姿态估计框架
- API稳定，文档完善
- 支持多种模型（RTMPose、HRNet等）
- 更精确的关键点检测

### 方案2：使用旧版MediaPipe

如果PyPI上有旧版本可用（需要验证）：

```bash
pip uninstall mediapipe
pip install 'mediapipe<0.10'  # 需要验证是否可用
```

### 方案3：更新代码以支持新API

需要重写`src/models/pose.py`以使用MediaPipe 0.10+的tasks API（需要一些时间）。

## 用户问题回答

**Q1：是否需要手动标注？**
A：**不需要**。系统会自动检测关键点并绘制骨架（就像图中的效果），完全自动化。

**Q2：MediaPipe是什么？**
A：MediaPipe是Google开发的人体姿态估计库，用于自动提取人体关键点。当前版本有API兼容性问题，建议使用MMPose替代。

**Q3：如何查看可视化效果？**
A：在姿态估计模块正常工作后，运行`scripts/visualize_pose_cm1_cm3.py`可以生成带有骨架线条的可视化图像（类似图中效果）。
