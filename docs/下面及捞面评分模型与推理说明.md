# 下面及捞面：关键帧评分 + 时间线性插值说明

## 流程概览

1. **人工评分关键帧**：在 Web「下面及捞面评分标注工具」中，对每个 xl 视频选出的关键帧（约 10 帧/视频）进行手动评分。
2. **非关键帧插值**：使用脚本基于关键帧时间做线性插值，为未评分的帧生成分数并写回 `*_scores.json`，并标记 `"interpolated": true`。

---

## 1. 关键帧与人工评分

- 关键帧列表：`data/scores/下面及捞面/{video}/key_frames.json`（由 `scripts/select_keyframes_boiling.py` 生成）。
- 人工评分结果：`data/scores/下面及捞面/{video}/{frame}_scores.json`，格式与 Web 保存一致；人工评分的帧无 `interpolated` 或 `interpolated: false`。

---

## 2. 非关键帧：时间线性插值

**命令：**

```bash
python scripts/interpolate_boiling_scores.py
```

- **逻辑**：对每个 xl 视频，仅处理「不在关键帧集合内」的帧。根据每帧的帧号，在前后关键帧之间对各项属性得分做**时间线性插值**（与抻面规则一致）：
  - 前后均有关键帧：线性插值；若关键帧间隔大于 150 帧，插值结果 ×0.8。
  - 仅前或仅后有关键帧：取该侧得分 ×0.9；若与当前帧间隔大于 150 帧，再 ×0.8。
- **输出**：在 `data/scores/下面及捞面/{video}/` 下为每帧生成或更新 `{frame}_scores.json`，结构与其他 scores 一致，并带 `"interpolated": true`，便于与人工评分区分。

**可选参数：**

- `--video xl1`：仅处理指定视频。
- `--overwrite-all`：覆盖所有非关键帧（默认仅覆盖此前为 inferred 或 interpolated 的 JSON）。

---

## 3. 依赖

- 仅需 Python 标准库与项目已有环境；无需 PyTorch 或评分模型。

---

## 4. 综合评分（骨架线 + 标注）

在完成关键帧评分及非关键帧插值后，可运行**综合评分**，将骨架线数据与标注数据结合（与抻面思路一致，但下面及捞面无标准序列、不计算 DTW）：

**命令：**

```bash
python scripts/process_all_boiling_scores.py
```

- **输入**：`data/scores/下面及捞面/hand_keypoints/hand_keypoints_xl*.json`（骨架线）、`data/scores/下面及捞面/{video}/*_scores.json`（标注，含关键帧与插值帧）。
- **逻辑**：仅对「有评分文件的抽取帧」评分；每帧根据视频帧号取对应骨架帧，提取手部角度特征得到 position/angle/coordination 骨架分，action 以标注为主；手部采用双置信度融合（骨架质量与标注置信度），面条/汤面与操作规范以标注聚合。
- **输出**：`reports/comprehensive_scores_boiling/comprehensive_score_{video}.json`、`summary.json`，格式与抻面报告兼容，供 Web 可视化使用。

**Web 可视化：** 首页 →「下面及捞面综合评分可视化」进入，或访问 `/scoring-visualization-boiling`，内容与布局仿照抻面综合评分（评分概览、检测质量、类别对比、趋势图等；无 DTW 图表）。

---

## 5. 与抻面的一致性

- 插值规则与抻面模块（`src/scoring/enhanced_comprehensive_scorer.py` 中的 `_interpolate_annotation_score`）一致：按帧号线性插值、单侧 0.9、关键帧间隔过大时 ×0.8（150 帧阈值）。
- 综合评分：手部融合方式与抻面类似（骨架 + 标注双置信度），下面及捞面因无标准动作序列不计算 DTW，动作分以标注为主，其余维度以标注聚合，在保证严谨的前提下简化实现。
