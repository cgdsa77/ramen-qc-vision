# 代码结构说明

## 当前阶段目标

**主要目标：训练检测模型，能够识别 hand, noodle_rope, noodle_bundle**

## 模块架构

### 1. 检测模块（主要功能）✅

**位置：** `src/models/detector.py`
- 使用YOLO进行目标检测
- 识别：hand, noodle_rope, noodle_bundle
- 当前状态：正在训练7个视频的模型

**相关文件：**
- `src/training/train_detection_model.py` - 训练脚本
- `src/api/video_detection_api.py` - 检测API
- `web/video_detection.html` - Web界面

### 2. 姿态估计模块（可选功能）✅

**位置：** `src/models/pose.py`
- 支持MediaPipe（已实现）
- 支持MMPose/RTMPose（预留接口）
- **独立运行，不影响检测功能**

**使用方式：**
```python
from src.models.pose import PoseEstimator
pose = PoseEstimator(cfg)
keypoints = pose.run_frame(frame)  # 获取关键点
```

**预留接口：**
- `PoseAnalyzer` - 姿态分析器（后续扩展）
- `ActionNormalityScorer` - 动作规范性评分器（后续扩展）

### 3. 粗细分析模块（预留接口）📋

**位置：** `src/models/thickness.py`
- `ThicknessEstimator` - 基于图像处理的粗细分析（基础实现）
- `ThicknessClassifier` - 分类器接口（预留）
- `ThicknessRegressionModel` - 回归模型接口（预留）

**当前状态：**
- 基础框架已创建
- 可以基于检测结果进行简单分析
- 高级功能预留接口

### 4. 评分模块（预留接口）📋

**位置：** `src/scoring/scorer.py`
- `Scorer` 类已创建，但主要功能预留
- 方便后续添加评分逻辑

### 5. Pipeline（流程编排）✅

**位置：** `src/pipelines/steps.py`
- `Pipeline` 类已优化，支持模块化运行
- 检测功能完整，其他模块可选

## 模块独立性

所有模块设计为**相互独立**：
- ✅ 检测模块可以独立运行
- ✅ 姿态估计可以独立运行（可选）
- ✅ 粗细分析基于检测结果，可选
- ✅ 评分模块为预留接口

## 扩展方式

### 添加新功能

1. **添加新的分析模块**
   - 在 `src/models/` 下创建新文件
   - 实现标准接口
   - 在 `Pipeline` 中集成

2. **扩展评分功能**
   - 在 `Scorer` 类中添加新方法
   - 使用检测、姿态、粗细等结果
   - 返回评分数据

3. **添加新的API接口**
   - 在 `src/pipelines/api_server.py` 中添加
   - 或创建新的API文件

## 当前不需要的功能

- ❌ 动作规范性评估（预留接口）
- ❌ 标准动作对比（预留接口）
- ❌ 综合评分报告（预留接口）
- ❌ 详细的分析报告生成（预留接口）

## 代码组织原则

1. **模块化**：每个功能独立成模块
2. **可选性**：非核心功能可以禁用
3. **可扩展**：为后续功能预留接口
4. **清晰性**：代码结构清晰，易于理解

## 后续扩展建议

当需要添加新功能时：
1. 在对应的预留接口类中实现
2. 或在对应模块中添加新方法
3. 不会影响现有功能
4. 保持模块独立性
