# 评分效果说明

## 当前评分结果

从您提供的评分结果来看，当前使用的是**占位规则**，而不是基于标准数据集的评分规则。

### 占位规则结果特征：
- `"rules_used": "占位规则:覆盖率+目标密度线性加权"`
- 评分范围：0-1之间的小数
- 评分方式：基于检测覆盖率和目标密度

### 标准评分规则结果特征（应该显示）：
- `"rules_used": "基于标准数据集的评分规则和阈值"`
- 评分范围：0-5分
- 评分方式：基于标准数据集的阈值和权重

## 可能的原因

1. **Web服务未重启**：如果Web服务在更新代码前就启动了，需要重启才能使用新代码
2. **评分规则加载失败**：可能评分规则文件路径有问题，导致回退到占位逻辑

## 解决方案

### 方法1：重启Web服务（推荐）

1. 停止当前Web服务（Ctrl+C）
2. 重新启动：
   ```bash
   python start_web.py
   ```
3. 重新测试评分功能

### 方法2：检查评分规则文件

确认评分规则文件存在：
```bash
# 检查文件是否存在
dir data\scores\抻面\scoring_rules.json
```

如果文件不存在，重新生成：
```bash
python scripts\build_scoring_rules.py
```
