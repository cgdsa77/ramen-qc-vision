# 检测模型训练说明（7个视频）

## 训练配置

### 数据集信息
- **训练视频**: cm1, cm2, cm3, cm4, cm5, cm6, cm7（共7个视频）
- **训练集图片**: 831张
- **验证集图片**: 56张（使用cm7作为验证集）
- **总图片数**: 887张
- **检测类别**: 
  - hand（手部）
  - noodle_rope（面条）
  - noodle_bundle（面条束）
  - pot_or_table（锅/桌子，如果已标注）

### 训练参数
- **训练轮数**: 150 epochs（比之前的50轮更多，有助于提升准确率）
- **批次大小**: 8（根据CPU内存调整）
- **图片尺寸**: 640x640
- **初始学习率**: 0.01
- **早停耐心值**: 50轮（如果50轮没有改善会自动停止）
- **设备**: CPU（避免CUDA兼容性问题）

### 数据增强
为了提高模型泛化能力，启用了以下数据增强：
- **色调/饱和度/明度调整**: HSV增强
- **旋转**: ±10度
- **平移/缩放**: 随机平移和缩放
- **左右翻转**: 50%概率
- **Mosaic增强**: 100%启用
- **MixUp增强**: 10%概率

## 训练命令

```bash
python src/training/train_detection_model.py --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 --epochs 150 --batch 8
```

### 可调整参数

如果需要进一步优化，可以调整以下参数：

```bash
# 增加训练轮数（如果150轮还不够）
python src/training/train_detection_model.py --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 --epochs 200 --batch 8

# 调整批次大小（如果内存不足可以减小）
python src/training/train_detection_model.py --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 --epochs 150 --batch 4

# 调整验证集比例（默认20%）
python src/training/train_detection_model.py --videos cm1 cm2 cm3 cm4 cm5 cm6 cm7 --epochs 150 --val-split 0.15
```

## 训练输出

训练完成后，模型会保存到：
- **最佳模型**: `models/stretch_detection/weights/best.pt`
- **最新模型**: `models/stretch_detection/weights/last.pt`
- **训练日志**: `models/stretch_detection/results.csv`
- **训练图表**: `models/stretch_detection/train_batch*.jpg`

## 训练后步骤

1. **等待训练完成**（可能需要30-60分钟，取决于CPU性能）

2. **重启Web服务器**以加载新模型：
   ```bash
   # 停止当前服务器（如果正在运行）
   # 然后重新启动
   python start_web.py
   ```

3. **测试检测效果**：
   - 在浏览器中打开：http://localhost:8000
   - 上传未标注的视频（如cm8, cm9, cm10等）进行测试
   - 查看检测准确率是否提升

4. **评估结果**：
   - 如果效果好：可以继续标注更多视频进一步提升
   - 如果效果一般：考虑增加训练轮数或检查标注质量
   - 如果效果差：检查数据标注是否正确、完整

## 预期改进

使用7个视频（vs之前的3个视频）进行训练，预期会有以下改进：

- ✅ **检测准确率提升**: 从约30-40%提升到60-75%
- ✅ **误检率降低**: 减少误检和漏检
- ✅ **边界框精度提升**: 检测框更准确
- ✅ **对未标注视频的泛化能力增强**: 在cm8-cm12等视频上表现更好

## 进一步优化建议

如果150轮训练后效果仍不够理想，可以考虑：

1. **增加训练轮数**: 200-300轮
2. **调整学习率**: 如果损失曲线不收敛，可以尝试更小的学习率（如0.005）
3. **检查标注质量**: 确保标注准确、完整、一致
4. **增加训练数据**: 继续标注更多视频（cm8, cm9, cm10等）
5. **使用更大的模型**: 如果CPU性能允许，可以尝试YOLOv8s或YOLOv8m（需要修改代码）
