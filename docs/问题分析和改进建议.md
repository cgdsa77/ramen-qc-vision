# 模型识别问题分析和改进建议

## 发现的问题

### 1. 类别配置问题 ✅ 已修复
- **问题**：`classes.txt`中定义了4个类别，但只标注了3个
- **影响**：模型学习了4个类别，但`pot_or_table`没有训练样本，导致类别预测混乱
- **解决**：已修改`classes.txt`，只保留3个已标注的类别

### 2. 类别分布不平衡
根据分析，7个视频的类别分布如下：
- **hand**: 857个 (35.1%)
- **noodle_rope**: 1119个 (45.8%) - 最多
- **noodle_bundle**: 469个 (19.2%) - 最少

**不同视频的分布差异很大**：
- cm1: hand 21.9%, noodle_rope 62.3%, noodle_bundle 15.8%
- cm2: hand 64.7%, noodle_rope 6.0%, noodle_bundle 29.4%
- cm4: hand 9.0%, noodle_rope 58.5%, noodle_bundle 32.4%
- cm6: 只有hand和noodle_rope，完全没有noodle_bundle

### 3. 识别混乱的可能原因

#### 3.1 hand与noodle_rope识别反了
**可能原因**：
- 在抻面动作中，手和面条在某些角度下外观相似（都是肤色/米色）
- 当手握住面条时，边界不清晰
- cm2中hand标注很多（64.7%），cm4中hand很少（9.0%），导致模型在不同视频上表现不一致

**建议**：
1. 检查标注质量，确保hand和noodle_rope的边界清晰
2. 补充更多hand和noodle_rope共存的样本
3. 考虑使用数据增强，特别是针对hand和noodle_rope容易混淆的场景

#### 3.2 标注数据量不足
- cm7只有56张图片，173个标注框（最少）
- cm1有86张，cm3有179张（最多）
- 数据量差异大，可能导致模型在数据少的视频上表现不佳
