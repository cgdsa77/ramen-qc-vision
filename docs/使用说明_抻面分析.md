# 抻面动作分析系统使用说明

## 系统概述

本系统基于已标注的标准动作视频（cm1, cm2, cm3）构建基准模型，用于分析新视频的抻面动作与标准动作的相似度。

## 工作流程

### 1. 训练基准模型（首次使用）

基于3个标准视频（cm1, cm2, cm3）训练基准模型：

```bash
python src/training/train_stretch_baseline.py
```

**输出：**
- 模型文件：`models/stretch_baseline/baseline_model.json`
- 训练信息会显示在控制台

**说明：**
- 模型从标注数据中提取特征（手部位置、面条位置、空间关系等）
- 建立标准动作的统计基准（均值、标准差等）
- 后续可以添加更多标准视频重新训练，提高模型准确性

### 2. 分析新视频

分析已标注的新视频（如 cm4, cm5 等）：

```bash
# 方式1：使用简单测试脚本（推荐）
python scripts/test_stretch_analysis_simple.py cm4

# 方式2：使用分析模块
python src/analysis/analyze_stretch.py --video cm4
```

**输出：**
- 相似度得分（0-1，越高越接近标准动作）
- 各项特征的详细得分
- JSON格式的分析报告：`reports/stretch_analysis_<视频名>.json`

## 分析结果说明

### 得分含义

- **0.9-1.0**: 优秀 ⭐⭐⭐ - 与标准动作高度一致
- **0.8-0.9**: 良好 ⭐⭐ - 基本符合标准
- **0.7-0.8**: 一般 ⭐ - 有改进空间
- **< 0.7**: 需要改进 - 与标准动作差异较大

### 关键特征

系统会分析以下关键特征：

1. **hand_presence_rate**: 手部检测率
2. **noodle_rope_presence_rate**: 面条检测率
3. **hand_x_mean / hand_y_mean**: 手部平均位置
4. **hand_to_rope_distance_mean**: 手部与面条的平均距离
5. **noodle_rope_x_mean**: 面条的平均位置

## 迭代改进流程

### 当前状态
- ✅ 基准模型：基于 cm1, cm2, cm3（3个标准视频）
- ⏳ 待分析：cm4, cm5, ..., cm12（9个视频）

### 改进步骤

1. **分析现有视频**
   ```bash
   python scripts/test_stretch_analysis_simple.py cm4
   ```

2. **评估分析效果**
   - 查看得分是否合理
   - 检查各项特征得分
   - 识别需要改进的地方

3. **标注更多标准视频**
   - 如果分析效果不理想，标注更多标准视频
   - 将新的标准视频加入训练集

4. **重新训练模型**
   - 修改 `train_stretch_baseline.py` 中的标准视频列表
   - 例如：`standard_videos = ['cm1', 'cm2', 'cm3', 'cm4']`
   - 重新运行训练脚本

5. **重复迭代**
   - 不断添加标准视频
   - 重新训练模型
   - 测试分析效果
   - 直到达到满意的准确度

## 注意事项

1. **标注要求**
   - 新视频必须先标注才能分析
   - 标注格式：YOLO格式（.txt文件）
   - 标注类别：hand, noodle_rope, noodle_bundle, pot_or_table

2. **模型更新**
   - 每次添加新的标准视频后，需要重新训练模型
   - 模型会学习新的标准动作模式
   - 更多标准视频 = 更准确的基准模型

3. **分析准确性**
   - 当前模型基于3个标准视频，准确性有限
   - 随着标准视频增加，准确性会提高
   - 建议至少标注5-10个标准视频后再正式使用

## 快速开始

```bash
# 1. 训练基准模型（首次使用）
python src/training/train_stretch_baseline.py

# 2. 分析新视频（假设cm4已标注）
python scripts/test_stretch_analysis_simple.py cm4

# 3. 查看结果
# 结果保存在 reports/stretch_analysis_cm4.json
```
