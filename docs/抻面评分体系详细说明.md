# 抻面部分评分体系详细说明

本文档描述当前抻面综合评分的完整逻辑、公式与配置，便于评估合理性并做优化。

---

## 一、总体架构

### 1.1 数据来源

| 来源 | 说明 |
|------|------|
| **骨架线数据** | MediaPipe 手部 21 关键点，存于 `data/scores/抻面/hand_keypoints/hand_keypoints_<视频>.json` |
| **关键帧标注** | 各视频目录下 `*_scores.json`，含 hand / noodle_rope / noodle_bundle 的人工评分 |
| **标准序列** | 仅从 **cm1、cm2、cm3** 三个视频提取的角度序列，用于 DTW 与关键点权重 |

### 1.2 评分流程概览

```
加载骨架线 → 提取角度序列 → 计算 DTW 得分（整段）
     ↓
逐帧：提取当前帧角度 → 手部骨架评分(position/angle/coordination) + action=DTW得分
     ↓
     若有关键帧标注：加载/插值得到 hand / noodle_rope / noodle_bundle 标注分
     ↓
     手部：骨架分与标注分按权重融合（并根据检测质量调权）
     面条/面条束：以标注为主，无标注时面条用默认 3.0 或骨架辅助
     ↓
按类别加权平均 → 动态权重（仅对存在的类别）→ 总分
```

### 1.3 核心代码与配置

- 评分器：`src/scoring/enhanced_comprehensive_scorer.py`（EnhancedComprehensiveScorer）
- 角度提取：`src/scoring/spatial_angle_extractor.py`
- DTW：`src/scoring/improved_dtw.py`
- 关键点权重：`src/scoring/keypoint_weight_estimator.py`（Mean Shift）
- 规则与阈值：`data/scores/抻面/scoring_rules.json`（当前综合评分中**未直接使用**该文件的阈值做分段，仅作参考）

## 二、评分类别与权重

### 2.1 类别权重（原始）

| 类别 | 权重 | 说明 |
|------|------|------|
| hand（手部） | 0.4 | 位置、动作、角度、协调性 |
| noodle_rope（面条绳） | 0.5 | 粗细、弹性、光泽、完整性 |
| noodle_bundle（面条束） | 0.1 | 紧实度、均匀度 |

**动态权重**：若某类别在该视频中**没有任何有效评分**（如无 noodle_bundle 标注），则该类别**不参与总分**，其余类别权重按比例重新归一化。例如只有 hand + noodle_rope 时，权重变为 hand 4/9、noodle_rope 5/9。

### 2.2 手部属性权重

| 属性 | 权重 | 含义 |
|------|------|------|
| position | 0.25 | 手部位置 |
| action | 0.30 | 动作规范性（整段 DTW 得分） |
| angle | 0.20 | 手掌/手腕角度 |
| coordination | 0.25 | 双手协调性 |

### 2.3 面条绳属性权重

| 属性 | 权重 | 含义 |
|------|------|------|
| thickness | 0.30 | 粗细 |
| elasticity | 0.25 | 弹性 |
| gloss | 0.20 | 光泽 |
| integrity | 0.25 | 完整性 |

### 2.4 面条束属性权重

| 属性 | 权重 |
|------|------|
| tightness | 0.5 |
| uniformity | 0.5 |

## 三、标准序列与 DTW

### 3.1 标准动作序列

- **来源**：仅 **cm1、cm2、cm3** 三个视频的 `hand_keypoints_*.json`。
- **内容**：对每帧从 21 手关键点提取多维角度特征（见空间角度提取），组成三条“标准角度序列”。
- **用途**：  
  - 与待评视频的整段角度序列做 DTW，得到**动作（action）评分**；  
  - 关键点权重由 `KeypointWeightEstimator` 基于这些标准序列的角变化统计 + Mean Shift 得到。

### 3.2 DTW 计算

- **输入**：待评视频角度序列 vs 三条标准序列（每条可能下采样到 `max_sequence_length=500`）。
- **距离**：加权 DTW（欧氏距离），权重来自 `keypoint_weights.json`。
- **匹配**：取与三条标准序列中**距离最小**的一条，记录 `min_distance` 和 `matched_sequence`（0/1/2 对应 cm1/cm2/cm3）。
- **归一化**：`normalized_distance = distance / max(len(test), len(std))`。
- **转为 0–5 分**：
  - 公式：`score = 5.0 * exp(-min_distance / 50.0)`，再裁剪到 [0, 5]。
  - 其中 **50.0** 为固定常数 `max_expected_distance`，未根据当前数据分布校准。

### 3.3 动作（action）评分

- **逐帧共用**：每一帧的 hand 评分里的 **action** 都取**同一段视频的 DTW 得分**（整段一个值），即所有帧的 action 相同。

## 四、手部评分详细逻辑

### 4.1 骨架线单帧评分（仅从角度）

- **position**：  
  - 依赖 `bilateral.wrist_connection_angle`（双手腕连线与水平夹角）。  
  - 理想角 0°（或 180°），偏离越大分越低：  
  - `score = 5 * (1 - min(1, deviation/90))`，无双手数据则为 0。

- **angle**：  
  - 依赖单手的 `palm_direction_angle`（手掌方向）。  
  - 理想 90°，同上线性衰减，无有效手则 0。

- **coordination**：  
  - 有双手：用 `palm_angle_difference`，差异越小分越高，公式同上思路。  
  - 仅单手：根据该手 `palm_direction_angle` 与 90° 的偏离给 2～4 分。  
  - 无任何手数据：固定 2.5 分。

- **action**：见上，整段 DTW 得分，每帧相同。

### 4.2 骨架与标注融合

对 position / angle / action / coordination 四个属性：

- **基础融合权重**（可被检测质量调整）：  
  - position: 骨架 0.4，标注 0.6  
  - angle: 0.5 / 0.5  
  - action: 0.6 / 0.4  
  - coordination: 0.5 / 0.5  

- **检测质量调整**：  
  - `quality_score = detection_rate * 0.6 + both_hands_rate * 0.4`（0~1）。  
  - 质量越低，骨架权重按比例缩小（最低约 0.3），标注权重相应提高；无质量信息则不调整。

- **融合公式**：  
  - 若该帧有该属性的**标注**：  
    `fused = skeleton * w_skeleton + annotation * w_annotation`  
  - 若无标注：  
    `fused = skeleton * 0.8`（仅骨架，且打 8 折）。

- 最终单属性裁剪到 [0, 5]。

### 4.3 手部类别得分

- 对每一帧得到 4 个属性融合分，再按属性权重 (0.25, 0.30, 0.20, 0.25) 加权平均得到该帧手部分。  
- 所有有效帧的手部分再平均，得到视频的 **hand_score**。

## 五、面条绳与面条束评分

### 5.1 面条绳（noodle_rope）

- **有标注**：直接使用关键帧标注的 thickness / elasticity / gloss / integrity；非关键帧通过**线性插值**得到各属性分（见下节插值）。
- **无标注**：  
  - 若有双手角度（如 `bilateral`）：四个属性均给**默认 3.0**。  
  - 若无双手数据：同样全部 3.0（无任何骨架推断）。
- 类别得分：各帧 noodle_rope 的四个属性按 (0.3, 0.25, 0.2, 0.25) 加权平均，再对帧平均，得到 **noodle_rope_score**。

### 5.2 面条束（noodle_bundle）

- **仅依赖标注**：tightness、uniformity；无标注则该帧无 noodle_bundle 分，且若全视频都无，则该类不参与总分（动态权重）。
- 类别得分：有标注的帧按 0.5/0.5 加权平均 tightness 与 uniformity，再对帧平均，得到 **noodle_bundle_score**。

## 六、关键帧与插值

### 6.1 标注加载

- 从 `data/scores/抻面/<视频名>/*_scores.json` 读取。
- 根据 detection 内字段判断类别：  
  - 含 position/action → hand  
  - 含 thickness/elasticity → noodle_rope  
  - 含 tightness/uniformity → noodle_bundle  
- 同一帧多框同类别时，该类别各属性取**平均**作为该关键帧的该类评分。

### 6.2 非关键帧插值

- 对当前帧号，在关键帧列表中找**前一个**和**后一个**关键帧。
- **前后都有**：该属性线性插值  
  `score = prev_score * (1 - alpha) + next_score * alpha`，其中 `alpha = (frame - prev) / (next - prev)`。
- **仅前有**：`score = prev_score * 0.9`。  
- **仅后有**：`score = next_score * 0.9`。  
- 若无任一侧关键帧，则该帧该属性无标注，手部用骨架分×0.8，面条/面条束按“无标注”规则处理。

---

## 七、总分计算

1. 按类别分别计算：**hand_avg_score**、**rope_avg_score**、**bundle_avg_score**（仅统计有数据的帧）。
2. **动态类别**：只保留得分列表非空的类别，记为 `available_classes`。
3. **动态权重**：对 `available_classes` 内类别，按原始类别权重重新归一化得到 `normalized_weights`。
4. **总分**：  
   `total_score = Σ (class_avg_score[cls] * normalized_weight[cls])`，对 `cls in available_classes`。

## 八、合理性简析

### 8.1 优点

- **多源融合**：骨架 + 关键帧标注，并有检测质量调节，减少低质量骨架的干扰。  
- **动态权重**：无 noodle_bundle 时不影响总分结构，符合“非所有视频都有面条束”的设定。  
- **标准序列 + DTW**：动作规范性有明确参考（cm1/cm2/cm3），可解释性强。  
- **关键点权重**：Mean Shift 基于标准序列学习角度维度权重，不是全等权。  
- **插值**：非关键帧有前后插值，避免仅关键帧有分、其余为 0 的断层。

### 8.2 不足与可优化点

1. **标准视频仅 3 个**  
   - 标准只来自 cm1、cm2、cm3，覆盖风格有限。  
   - **可优化**：将 cm4、cm5 等质量高的已标注视频纳入标准序列；或按“多模板”取最小 DTW 的现有逻辑扩展标准集。

2. **DTW 转分数的常数固定**  
   - `max_expected_distance = 50.0` 与 `score = 5*exp(-d/50)` 未根据实际距离分布校准，可能造成多数视频集中在某一分数段。  
   - **可优化**：用标准视频与一批待评视频的 DTW 距离分布，估计分位数或均值/方差，再设 `max_expected_distance` 或改用分段/线性映射。

3. **action 全帧相同**  
   - 整段一个 DTW 分作为每帧的 action，无法体现“某一小段动作更差”。  
   - **可优化**：滑窗或分段 DTW，得到每段/每帧的 action 分再插值到帧；或保留整段 action 但明确其表示“整体动作规范性”。

4. **面条无骨架推断**  
   - 无标注时面条绳固定 3.0，面条束不参与；骨架信息未用于推断面条质量。  
   - **可优化**：用双手距离、手腕间距等粗粒度特征做简单回归或分级（仅作辅助），并限制默认分使用范围（如仅缺标注时使用）。

5. **scoring_rules.json 未参与综合分**  
   - 文档与文件中虽有 excellent/good/fair/poor 等阈值，但当前综合评分逻辑中**未**用这些阈值做分段或映射。  
   - **可优化**：若希望与“标准数据集 36 关键帧”一致，可将标注分或骨架分按这些阈值映射到等级再参与融合；或明确仅作展示/报告用。

6. **关键帧密度不均**  
   - 不同视频关键帧数量与间隔差异大，插值误差不一。  
   - **可优化**：对关键帧稀疏的视频降低标注权重或给出“置信度”提示；或建议最少关键帧数/最大间隔。

7. **检测质量公式简单**  
   - `quality_score = 0.6*detection_rate + 0.4*both_hands_rate`，未考虑连续缺失、抖动等。  
   - **可优化**：加入连续缺失长度、稳定性等，或对低质量段降权而非整段统一调权。

8. **关键点权重仅从标准视频学习**  
   - 若 cm1/cm2/cm3 动作风格单一，学到的权重可能偏向某类动作。  
   - **可优化**：加入更多样化的标准序列或正则化，避免某几个角度权重过高。

---

## 九、总结

当前抻面评分体系是**以手部骨架角度 + 关键帧标注为主、面条以标注为主、整段 DTW 作为动作规范性**的综合方案；通过动态类别权重和检测质量调节，在“部分视频缺面条束”“骨架质量差”等情况下有一定鲁棒性。  

若要进一步提升合理性与可解释性，建议优先：  
- 校准 DTW 距离到分数的映射；  
- 将标准序列扩展或分段 DTW 细化 action；  
- 明确是否采用 `scoring_rules.json` 的阈值并统一到综合分计算中；  
- 在无标注时对面条做有限的骨架辅助推断并约束默认分使用范围。
