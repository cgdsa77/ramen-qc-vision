# 兰州拉面质检系统 — 主登录界面展示说明文档

本文档描述当前**主登录界面**的结构、样式与交互，供其他 AI 或开发者在不改动登录逻辑与接口的前提下进行视觉与动效优化。优化时请勿修改：登录接口调用、会话校验、`showMain()` 显隐逻辑、表单校验与错误提示逻辑。

---

## 一、界面定位与触发条件

- **系统名称**：面向兰州拉面制作的标准化流程视觉质检系统  
- **主登录界面**：未登录或会话失效时，页面以登录态展示；用户看到的是居中登录弹窗 + 背景（含左右侧栏）。  
- **状态标识**：`body` 带有 `login-page` 类时表示当前为登录页；`#loginModal` 无 `hidden` 类时登录弹窗显示。  
- **与主界面关系**：登录成功后执行 `showMain(true)`，会为 body 加上 `main-page` 和 `role-*`、去掉 `login-page`，并隐藏 `#loginModal`、显示 `#mainWrap`。本说明仅针对登录页展示与动效，不涉及登录后主界面。

---

## 二、技术栈与文件位置

- **唯一入口文件**：`web/index.html`（内联 CSS + 内联 JS，无前端框架）  
- **字体**：Google Fonts — Noto Serif SC、Noto Sans SC（与主界面一致）  
- **动效**：Popmotion 已集成（`/web/js/popmotion.bundle.js`，通过 `window.popmotion` 使用）  
- **后端**：FastAPI（`start_web.py`），登录接口 `POST /api/auth/login`，会话 `GET /api/auth/session`

---

## 三、登录页整体布局（自上而下 / 层级）

1. **背景层**  
   - `body.login-page` 使用整页背景：  
     - `linear-gradient(rgba(44,24,16,0.35), rgba(44,24,16,0.35))` 叠在  
     - `url('/data/web_background_images/web1_bg.jpg') center center / contain no-repeat fixed`  
   - 效果：暖色半透明遮罩 + 背景图（建筑物/水景等），整体偏暗以突出前景。

2. **左右侧栏（播放带）**  
   - 仅登录页显示：`body.login-page .side-panel { display: flex }`  
   - 左侧：`.side-panel.left`；右侧：`.side-panel.right`  
   - 每侧为多列 `.side-strip`，内为 `.side-strip-track`，用于展示拉面相关图片条；默认 `filter: brightness(0.42)`，hover 时 `brightness(1)`。  
   - 优化时可考虑：模糊强度、宽度、圆角、与背景的层次关系；**不可删除或改变其显示条件**，以免破坏布局。

3. **登录弹窗（核心）**  
   - 容器：`#loginModal.modal-overlay`  
     - `position: fixed; inset: 0; background: rgba(0,0,0,0.5); z-index: 9999`  
     - `display: flex; align-items: center; justify-content: center; padding: 20px`  
   - 内容框：`#loginModal .modal-box`  
     - 白色背景 `var(--warm-white)`（#fffbf7）、圆角 8px、阴影 `var(--shadow)`、`max-width: 400px`、`padding: 28px`  
     - 无关闭按钮，必须通过登录或会话进入主界面

## 四、登录弹窗内部结构（DOM 与文案）

| 顺序 | 元素 | 说明 |
|------|------|------|
| 1 | `<h3>系统登录</h3>` | 标题，居中，当前样式见下文 |
| 2 | `.form-group` | 第一组：`<label>登录账号</label>` + `<input id="loginUsername" placeholder="请输入登录账号" autocomplete="username">` |
| 3 | `.form-group` | 第二组：`<label>密码</label>` + `<input id="loginPassword" type="password" placeholder="请输入密码" autocomplete="current-password">` |
| 4 | `#loginBtn.btn-primary` | 登录按钮，内有两段：`.login-btn-text`（“登录”）与 `.login-btn-spinner`（加载时显示） |
| 5 | `#loginError.error-msg` | 错误信息，默认 `display: none`，校验或接口报错时显示 |
| 6 | `.hint` | 文案：“忘记密码？联系管理员重置” |

**注意**：  
- 登录逻辑使用 `#loginUsername`、`#loginPassword`、`#loginBtn`、`#loginError`，优化时请保留这些 id 与表单项的对应关系，以及按钮的 `id="loginBtn"` 与 `class="btn-primary"`（含 `is-loading` 的加载态）。  
- “忘记密码？联系管理员重置” 目前仅为提示文案，无跳转或接口，可改样式与排版，不建议改成可点击链接（除非产品明确要求）。

## 六、当前动效与交互（JS）

- **弹窗入场**：`runLoginAnimations()` 中对 `#loginModal .modal-box` 做：  
  - 初始：`opacity: 0`，`transform: translateY(-50px)`  
  - 使用 Popmotion spring（stiffness: 45, damping: 14）将 opacity 0→1、translateY -50→0  
  - 调用时机：切换到登录页时（如 `showMain(false)` 后约 50ms）执行 `runLoginAnimations()`。  
- **输入框 focus**：在 `bindLoginAnimations()` 中为 `#loginUsername`、`#loginPassword` 绑定 focus/blur：  
  - focus：`transform: translateY(-2px)`，`box-shadow: 0 4px 12px rgba(196,92,38,0.2)`  
  - blur：恢复为空  
- **登录按钮**：同一处为 `#loginBtn` 绑定 mouseenter/mouseleave/mousedown/mouseup，使用 Popmotion 做 scale(1 ↔ 1.03) 与按下 0.97。  

优化时可调整弹簧参数、入场方向/距离、输入框 focus 的位移与阴影、按钮 hover/按下幅度；**请勿删除**对 `#loginModal .modal-box`、`#loginUsername`、`#loginPassword`、`#loginBtn` 的引用与事件绑定，以免破坏现有动效或后续逻辑。

## 八、建议优化方向（不改变硬逻辑）

以下仅为建议，实现时需保持上述 id/class 与登录流程不变。

1. **视觉**  
   - 模态框：圆角大小、阴影强度、与背景的对比度。  
   - 按钮：渐变方向、色值、hover 时的渐变位移或内发光（可与登录后主界面卡片按钮风格统一）。  
   - 输入框：边框色、圆角、占位符颜色、focus 时的边框/阴影/背景。

2. **背景与层次**  
   - 背景图与遮罩的透明度、模糊程度（若使用 `backdrop-filter` 需注意兼容性）。  
   - 侧栏与背景、与弹窗的层级与对比度。

3. **动效**  
   - 弹窗入场：改为 scale + opacity、或不同方向的 translate，并保持 Popmotion spring 调用。  
   - 输入框：focus 时的过渡曲线、位移与阴影强度。  
   - 按钮：hover/active 的 scale 或轻微位移，与现有 `is-loading` 动画兼容。

4. **响应式与无障碍**  
   - 小屏下弹窗宽度、内边距、字号。  
   - 为登录按钮、输入框增加合适的 `aria-label` 或说明，保持对比度满足可读性。

5. **字体与排版**  
   - 标题与标签的字重、字间距、行高，与 Noto 字体搭配一致。

---

## 九、修改执行规则

1. **不得修改**：  
   - 登录接口调用方式、请求体、成功/失败分支；  
   - `showMain()`、会话检查、`#loginModal` / `#mainWrap` 的显隐逻辑；  
   - `#loginUsername`、`#loginPassword`、`#loginBtn`、`#loginError` 的 id 及表单与按钮的语义关系；  
   - 登录按钮的 `is-loading` 行为及 `.login-btn-spinner` 的展示逻辑。

2. **可修改**：  
   - 仅限 `web/index.html` 内与登录页相关的 CSS、以及登录弹窗/背景/侧栏的样式与动效参数；  
   - 可在不破坏上述 id 与事件绑定的前提下，调整 `runLoginAnimations` / `bindLoginAnimations` 中的动画参数或增加新动画。

3. **优化后须保证**：  
   - 登录流程与错误提示行为与现在一致；  
   - 登录页在常见分辨率下布局正常、无错位或遮挡；  
   - 动效流畅、无卡顿，且不阻塞点击与输入。
